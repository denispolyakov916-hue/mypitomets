# Правила разработки

## Git Flow

### Ветки

- `main` — стабильная версия
- `feature/{module}-{desc}` — новая функциональность
- `fix/{module}-{desc}` — исправление бага
- `refactor/{desc}` — рефакторинг

### Коммиты

На русском языке с префиксами:

```
feat: добавлена фильтрация товаров по цене
fix: исправлено отображение аватара в профиле
refactor: вынесена логика расчёта калорий в отдельный сервис
docs: обновлена спецификация модуля авторизации
test: добавлены тесты для калькулятора калорий
chore: обновлены зависимости backend
```

## Code Review

### Чеклист перед PR

- [ ] Код следует конвенциям проекта (см. `.cursor/rules/`)
- [ ] Бизнес-логика в сервисном слое, не во views
- [ ] Добавлены/обновлены тесты
- [ ] Нет секретов в коде (пароли, ключи)
- [ ] Миграции созданы и применяются корректно
- [ ] API формат соответствует стандарту (data/error/code)

## Стиль кода

Подробные правила — в `.cursor/rules/code-style.mdc`.

### Основное

- Комментарии и docstrings — на русском
- Имена переменных и функций — на английском
- Python: snake_case, JS: camelCase/PascalCase
- Импорты: стандартные → сторонние → локальные

## Зависимости

- Backend: `pip install package==version` → добавить в `requirements.txt` с pinned версией
- Frontend: `npm install package` → автоматически в `package.json`
