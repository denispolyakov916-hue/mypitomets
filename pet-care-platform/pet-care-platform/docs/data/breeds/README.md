# Справочник пород

## Назначение

Справочник пород используется в PetID, подборе питания, персонализации товаров, подсказках по уходу и аналитике.

Справочник не должен описываться как "все породы мира", пока для этого нет полного отчета покрытия по выбранным реестрам. Корректная формулировка: "справочник пород собак и кошек, сверяемый с официальными кинологическими и фелинологическими реестрами".

## Где лежат данные

Текущие файлы данных:

| Файл | Назначение |
|------|------------|
| `docs/data/breeds/breeds.json` | Основной справочник пород. |
| `docs/data/breeds/breed_health.json` | Данные по здоровью и породным рискам. |
| `docs/data/breeds/breed_nutrition.json` | Данные для питания и рекомендаций. |
| `docs/data/breeds/breed_care.json` | Данные по уходу. |
| `docs/data/breeds/common_data.json` | Общие справочные данные. |
| `docs/data/nutrition/breeds.json` | Дополнительный справочник для питания собак. |
| `docs/data/nutrition/cats_breeds.json` | Дополнительный справочник для питания кошек. |
| `frontend/src/data/breeds.js` | Frontend-данные или fallback-данные, если используются в UI. |

Текущие модели backend:

- `backend/apps/pets/breed_models.py`
- `backend/apps/pets/models.py`

Текущие импортеры:

- `backend/apps/pets/management/commands/load_breeds.py`
- `backend/core/management/commands/import_breeds.py`

## Основные сущности

### Breed

Справочник породы.

Ключевые поля:

| Поле | Назначение |
|------|------------|
| `id` | Внутренний идентификатор. |
| `external_id` | Внешний идентификатор, если используется импорт. |
| `species` | `dog` или `cat`. |
| `name` | Название на русском. |
| `name_en` | Название на английском. |
| `slug` | URL-совместимый идентификатор. |
| `fci_number`, `fci_group`, `fci_section` | Поля классификации FCI для собак. |
| `wcf_code`, `wcf_category` | Поля WCF для кошек, если применимо. |
| `country_origin` | Страна происхождения. |
| `purpose` | Назначение породы. |
| `size_category` | Размерная категория. |
| `coat_type` | Тип шерсти. |
| `average_lifespan` | Средняя продолжительность жизни. |
| `weight_male_min`, `weight_male_max` | Вес самцов. |
| `weight_female_min`, `weight_female_max` | Вес самок. |
| `base_activity_level` | Базовая активность. |
| `trainability` | Обучаемость. |
| `grooming_needs` | Потребность в уходе. |
| `health_risks` | JSON с рисками здоровья. |
| `allergy_risks` | JSON с аллергическими рисками. |

### BreedHealth

Дополнительные данные о здоровье породы.

Примеры данных:

- породные риски;
- профилактические проверки;
- рекомендации по наблюдению;
- ссылки на источники.

### BreedNutrition

Дополнительные данные для питания.

Примеры данных:

- склонность к набору веса;
- чувствительность пищеварения;
- особенности рациона;
- ограничения для рекомендаций.

### BreedCare

Данные по уходу.

Примеры данных:

- груминг;
- активность;
- условия содержания;
- частота процедур.

## Целевая схема источников

В текущих JSON-файлах не хватает обязательных полей доказуемости для каждой записи. Целевое состояние:

| Поле | Обязательность | Описание |
|------|----------------|----------|
| `source_organization` | Да | FCI, AKC, FIFe, TICA, WCF, CFA или другой источник. |
| `source_title` | Да | Название стандарта, страницы или документа. |
| `source_url` | Да | Прямая ссылка на источник. |
| `source_accessed_at` | Да | Дата последней проверки. |
| `registry_status` | Да | `recognized`, `provisional`, `experimental`, `not_recognized`, `mixed`, `unknown`. |
| `data_confidence` | Да | `high`, `medium`, `low`, `unverified`. |
| `notes` | Нет | Комментарий по спорным моментам. |

## Источники для собак

Основной источник:

- FCI Breed Nomenclature: https://www.fci.be/en/nomenclature/

Дополнительная сверка:

- AKC Dog Breeds: https://www.akc.org/dog-breeds/
- The Royal Kennel Club Breeds A-Z: https://www.royalkennelclub.com/search/breeds-a-to-z/
- OFA CHIC Browse by Breed: https://ofa.org/chic-programs/browse-by-breed/

## Источники для кошек

Основные и дополнительные реестры:

- FIFe Breeds: https://fifeweb.org/cats/breeds/
- TICA: https://tica.org/
- WCF: https://wcf.de/en/
- CFA Breeds: https://cfa.org/breeds/

У кошек разные организации признают разные наборы пород. Поэтому в базе нужно хранить не только породу, но и конкретный реестр, которым она подтверждена.

## Правила заполнения

1. `species + slug` должны быть уникальны.
2. Для собак FCI-поля заполняются только если порода признана или описана FCI.
3. Для кошек WCF/FIFe/TICA/CFA-поля не должны смешиваться без указания реестра.
4. Вес должен храниться отдельно для самцов и самок, если источник это поддерживает.
5. Health-риск должен быть справочным предупреждением, а не диагнозом.
6. Для беспородных и метисов должна быть отдельная логика, а не фиктивная порода.
7. Нельзя использовать маркетинговые описания как медицинский источник.

## Отчет покрытия

Для доказуемости справочника нужен отдельный отчет покрытия:

| Метрика | Что показывает |
|---------|----------------|
| Всего пород собак | Количество dog-записей. |
| Всего пород кошек | Количество cat-записей. |
| Подтверждены источником | Количество записей с `source_url`. |
| Проверены за последние 12 месяцев | Количество записей со свежей датой проверки. |
| Высокий уровень доверия | Количество записей с `data_confidence = high`. |
| Требуют проверки | Записи без источника или с `unverified`. |
| Дубли | Совпадения по названию, alias или slug. |

Отчет должен обновляться после каждого изменения JSON-файлов или импорта пород.

## Проверки перед импортом

Перед загрузкой справочника нужно проверить:

- валидность JSON;
- обязательные поля;
- уникальность `species + slug`;
- отсутствие пустых названий;
- корректность `species`;
- корректность ссылок источников;
- отсутствие заявлений без источника;
- совместимость с backend-моделью `Breed`.

## Ограничения

Справочник пород не является ветеринарной базой диагнозов. Он может использоваться для персонализации и справочных подсказок, но не должен заменять консультацию специалиста.
