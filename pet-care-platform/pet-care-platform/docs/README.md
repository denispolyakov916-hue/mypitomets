# Документация проекта Питомец+

Дата ревизии: 2026-06-30

## Назначение

Эта папка содержит рабочую документацию проекта Питомец+. Документация нужна для владельца продукта, команды разработки, тестировщиков, администраторов и будущих партнеров.

Главные принципы:

- описывать фактическое состояние проекта, а не желаемое;
- отделять реализованное от планируемого;
- не делать медицинских или справочных утверждений без источника;
- поддерживать критические пользовательские сценарии как регрессионную базу;
- обновлять документацию при изменении моделей, API, алгоритмов и справочников.

## Конвенции документирования

### Структура модуля

Каждый модуль в `modules/` должен содержать:

- `specification.md` - техническая спецификация;
- `user-stories.md` - пользовательские истории и критерии приемки;
- дополнительные документы, если модуль содержит алгоритмы, справочники, интеграции или админские сценарии.

### Формат specification.md

```markdown
# Модуль: Название

## Версия и статус

- Версия:
- Статус:
- Последняя сверка с кодом:

## Назначение

## Зоны кода

## API

## Модели данных

## Бизнес-логика

## Ограничения

## Тестирование
```

### Формат user-stories.md

```markdown
# User Stories: Название модуля

## US-XX: Название истории

- Как: [роль]
- Хочу: [действие]
- Чтобы: [ценность]

### Критерии приемки

- [ ] Критерий 1
- [ ] Критерий 2
```

## Правила актуализации

Документацию нужно обновлять, если:

- изменился API;
- изменилась модель данных или миграция;
- изменился пользовательский сценарий;
- изменился алгоритм питания или рекомендаций;
- добавлен новый источник данных;
- исправлен критический баг;
- добавлена новая роль или админская функция.

## Архив

Старая документация находится в `docs_old/`. Она используется только как исторический источник. Перед переносом любого фрагмента в `docs/` его нужно проверить по актуальному коду.
