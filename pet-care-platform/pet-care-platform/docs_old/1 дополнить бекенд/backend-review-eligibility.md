
## Требование для бекенда: доступ к форме отзыва только после доставки

### Контекст
Во фронтенде форма отзыва должна быть доступна **только** пользователям, у которых есть заказ с этим товаром/курсом в статусе **"доставлен"**. До этого момента — только просмотр и фильтрация отзывов.

### Текущее поведение
- Фронтенд вызывает endpoint проверки доступности:
  - `GET /reviews/products/{id}/eligibility/`
  - `GET /reviews/courses/{id}/eligibility/`
- Результат используется как единственный источник истины для показа формы отзыва.

### Новое поведение (обязательно)
Endpoint `eligibility` должен возвращать `can_review = true` **только если**:
1) У пользователя есть заказ с данным товаром/курсом  
2) Статус соответствующего заказа = `delivered`  

Во всех остальных случаях `can_review = false`.

### Предлагаемый контракт ответа
```json
{
  "can_review": false,
  "reason": "order_not_delivered",
  "delivered_at": null,
  "order_id": null
}
```

Возможные `reason`:
- `not_authenticated`
- `no_purchase`
- `order_not_delivered`
- `already_reviewed` (если пользователь уже оставил отзыв)
- `ok`

### Дополнительно (желательно)
Если пользователь уже оставил отзыв, возвращать:
```json
{
  "can_review": false,
  "reason": "already_reviewed",
  "review_id": 123
}
```

### Изменения на стороне фронтенда
Фронтенд уже отключил fallback‑логику по `isPurchased`.

## Требование для бекенда: фото товаров из сервиса Kotmatros

### Контекст
Фото товаров должны подтягиваться **не из локального хранилища**, а из внешнего сервиса магазина (Kotmatros). В данных Kotmatros уже есть актуальные `url` изображений, пока используем их как источник истины.

### Новое поведение (обязательно)
- Бекенд при отдаче списка товаров и карточки товара должен возвращать **url изображений**, полученных из Kotmatros.
- Эти url используются фронтендом напрямую для рендера карточек и детальной страницы.

### Что требуется дореализовать на бекенде
1) Интеграция получения/сопоставления изображений из данных Kotmatros.
2) Возврат изображений в API товаров в согласованном формате.

### Предлагаемый контракт ответа (пример)
```json
{
  "id": 123,
  "name": "Название товара",
  "images": [
    "https://kotmatros.ru/wa-data/public/shop/products/77/94/69477/images/161750/161750.970.jpg",
    "https://kotmatros.ru/wa-data/public/shop/products/77/94/69477/images/161751/161751.970.jpg"
  ],
  "image": "https://kotmatros.ru/wa-data/public/shop/products/77/94/69477/images/161750/161750.970.jpg"
}
```

### Примечания
- Если `images` пустой, фронтенд покажет плейсхолдер, поэтому важно корректно заполнить массив.
- На первом этапе можно возвращать только `images` (а `image` взять как `images[0]`).
