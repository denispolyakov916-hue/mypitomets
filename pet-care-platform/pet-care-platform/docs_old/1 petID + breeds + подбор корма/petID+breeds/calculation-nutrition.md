## 1. Описание

### 1.1 Назначение функции

Универсальная функция `calculate_daily_calories()` предназначена для расчета стабильной дневной калорийности рациона питания домашних животных (кошек и собак) на основе индивидуальных параметров питомца, его физиологического состояния, уровня активности и внешних факторов.

**Функция обеспечивает:**
- **Персонализированный расчет** калорийности на основе веса, возраста, породы и здоровья животного
- **Динамическое планирование** на 14 дней с учетом ежедневных активностей
- **Адаптацию к изменениям** в расписании и состоянии питомца
- **Валидацию данных** на соответствие ветеринарным нормам

### 1.2 Сертификация и источники данных

Все используемые коэффициенты и формулы основаны на рекомендациях авторитетных ветеринарных организаций:

## 2. Логика работы функции

### 2.1 Основной алгоритм

**Для собак (формула NRC/FEDIAF):**

```
RER = 70 × (вес_в_кг)^0.75  # Resting Energy Requirement (ккал/день)
MER = RER × K_base × K_health × K_activity_extra  # Maintenance Energy Requirement
```

**Для кошек (формула FEDIAF 2021):**

```
RER = 70 × (вес_в_кг)^0.67  # Для кошек используется экспонента 0.67
MER = RER × K_base × K_health × K_activity_extra
```

**Где:**
- `K_base` — базовый коэффициент (возраст × стерилизация × активность × содержание × шерсть)
- `K_health` — коэффициент здоровья (рассчитывается по матрице заболеваний)
- `K_activity_extra` — дополнительные калории за конкретные активности

### 2.2 Двухнедельное планирование

Функция рассчитывает индивидуальный рацион на 14 дней с учетом:

**Статические параметры (не меняются в течение периода):**
- Возраст, порода, вес
- Стерилизация/кастрация
- Хронические заболевания
- Тип шерсти

**Динамические параметры (могут меняться ежедневно):**
- Запланированные активности (прогулки, бег, игры)
- Климатические условия (температура дня)
- Стресс-факторы

**Ежедневные активности (ввод пользователя):**

| Параметр | Тип данных | Пример |
|----------|------------|--------|
| Тип активности | enum | walk, run, play, swim, training |
| Длительность | int (минуты) | 30 |
| Интенсивность | enum | low, medium, high |
| Время суток | enum | morning, afternoon, evening |

### 2.3 Адаптивная логика

- **Базовый расчет**: MER по статическим параметрам (формула выше)
- **Динамическая надбавка**: + ккал за каждую запланированную активность
- **Валидация**: Итоговая калорийность должна быть в диапазоне 0.7×MER - 1.5×MER
- **Предупреждения**: При выходе за диапазон — рекомендация консультации ветеринара

### 2.4 Порядок применения коэффициентов

```
1. Рассчитать RER по весу
2. Определить K_base (возраст × стерилизация × активность × климат × содержание × шерсть)
3. Определить K_health по матрице заболеваний
4. Рассчитать базовый MER = RER × K_base × K_health
5. Добавить калории за активности: MER_final = MER + Σ(activity_calories)
6. Валидация и корректировка
```

### 2.5 Обработка множественных заболеваний

**Проблема:** У питомца может быть несколько заболеваний одновременно, которые по-разному влияют на калорийность:
- Одни требуют **снижения** калорий (ожирение, диабет)
- Другие требуют **повышения** калорий (гипертиреоз, недобор веса)
- Некоторые **конфликтуют** между собой

**Решение: Приоритетная система с матрицей взаимодействий**

**Шаг 1. Классификация заболеваний по приоритету:**

| Приоритет | Описание | Примеры |
|-----------|----------|---------|
| **CRITICAL (1)** | Угроза жизни | ХБП 3-4 стадии, острая печеночная недостаточность |
| **HIGH (2)** | Серьезное влияние на метаболизм | Сахарный диабет, гипертиреоз, ожирение 3 степени |
| **MEDIUM (3)** | Умеренное влияние | Ожирение 1-2 степени, гипотиреоз, ХБП 1-2 стадии |
| **LOW (4)** | Минимальное влияние на калорийность | Аллергии, артрит, заболевания кожи |

**Шаг 2. Определение направления влияния:**

| Направление | Коэффициент | Заболевания |
|-------------|-------------|-------------|
| **DECREASE** | < 1.0 | Ожирение, диабет, ХБП, болезни печени, гипотиреоз |
| **INCREASE** | > 1.0 | Гипертиреоз, недобор веса, восстановление после операции |
| **NEUTRAL** | = 1.0 | Аллергии, артрит (влияют на состав, не на калории) |

**Шаг 3. Валидация коэффициентов:**

> **Важно:** Разные коэффициенты имеют разные допустимые диапазоны!

| Коэффициент | Норма | Предупреждение | Критично |
|-------------|-------|----------------|----------|
| K_health (заболевания) | 0.7 - 1.3 | 0.5-0.7 или 1.3-1.5 | < 0.5 |
| K_age (возраст) | 1.0 - 3.0 | - | < 0.8 или > 3.2 |
| K_reproductive (репродукция) | 1.0 - 3.5 | - | > 4.0 |
| K_activity (активность) | 0.9 - 2.0 | - | > 2.5 |

**Валидация итогового MER (контекстная):**

| Диапазон MER/RER | Контекст | Статус | Действие |
|------------------|----------|--------|----------|
| 0.6 - 0.8 | Есть ожирение 2-3 ст. | ✅ Норма | Программа похудения |
| 0.6 - 0.8 | Нет ожирения | 🚫 Критично | Ошибка данных, требуется проверка |
| < 0.6 | Любой | 🚫 Критично | Слишком сильное ограничение, риск здоровью |
| 0.8 - 1.0 | Есть ожирение 1 ст. | ✅ Норма | Умеренное похудение |
| 0.8 - 1.0 | Нет ожирения | ⚠️ Предупреждение | Проверить данные |
| 1.0 - 4.0 | Любой | ✅ Норма | Стандартный расчет |
| 4.0 - 5.0 | Рост + лактация | ✅ Норма | Экстремальная потребность |
| 4.0 - 5.0 | Без роста/лактации | ⚠️ Предупреждение | Проверить данные |
| > 5.0 | Любой | 🚫 Критично | Ошибка данных |

### 3.2 Принцип денормализации

> **Важно:** PetID содержит все необходимые параметры для расчета калорийности. 
> Данные из базы знаний пород копируются в PetID при создании/обновлении питомца.
> Это позволяет избежать JOIN при расчете и ускоряет работу системы.

**Преимущества:**
- Один запрос к `pet` для получения всех данных расчета
- Нет зависимости от изменений в базе пород при расчете
- Пользователь может переопределить любые значения

**Этапы создания PetID:**
1. **Этап 1 (Базовый профиль):** name, species, breed_id, date_of_birth, sex, weight_kg, is_neutered
2. **Триггер автозаполнения:** size_category, coat_type, ideal_weight_kg, activity_level, body_condition_score
3. **Этап 2 (Расширенный профиль):** Пользователь может уточнить/переопределить любые поля

> **📄 Подробнее:** См. [Integration_PetID_Breeds_Calculator.md](../02.1%20Маппинг%20данных%20(PetID%20+%20Breeds)/Integration_PetID_Breeds_Calculator.md)

### 3.2.1 Принцип расчёта по неполным данным

**Калькулятор ВСЕГДА работает**, даже с минимальными данными.

| Данные | Для расчёта | Если NULL |
|--------|-------------|-----------|
| `species` | Выбор формулы | ❌ Расчёт невозможен |
| `weight_kg` | RER = 70 × BW^0.75 | ❌ Расчёт невозможен |
| `date_of_birth` | K_age | Принимаем "взрослый" |
| `is_neutered` | K_base | Принимаем false |
| `activity_level` | K_activity | Принимаем "moderate" |
| `size_category` | K_size | Рассчитываем по весу |
| `health_conditions` | K_health | K_health = 1.0 |
| `reproductive_state` | K_reproductive | K_reproductive = 1.0 |

**Уведомление при неполных данных:**
```
⚠️ Расчёт выполнен на основе имеющихся данных.
   Для повышения точности заполните профиль питомца полностью.
   
   Наши рекомендации основаны на передовых ветеринарных данных.
   Для детализации плана питания рекомендуем консультацию ветеринара.
```

#### Физические параметры

| Поле | Тип | Обязат. | Ограничения | Описание |
|------|-----|---------|-------------|----------|
| birth_date | date | да | <= today | Дата рождения → K_age |

### 6.4 Формула расчета

```python
def calculate_daily_nutrients(calories: float, species: str, life_stage: str) -> dict:
    """
    Рассчитывает суточную потребность в нутриентах.
    
    Args:
        calories: Суточная калорийность (MER)
        species: 'dog' или 'cat'
        life_stage: 'maintenance', 'growth', 'reproduction'
    
    Returns:
        dict с потребностями в нутриентах
    """
    # Нормализуем на 1000 ккал
    factor = calories / 1000
    
    # Получаем профиль нутриентов для вида и стадии
    profile = get_nutrient_profile(species, life_stage)
    
    return {
        'protein_g': profile.protein_min * factor,
        'fat_g': profile.fat_min * factor,
        'fiber_g': profile.fiber_recommended * factor,
        'calcium_g': profile.calcium_min * factor,
        'phosphorus_g': profile.phosphorus_min * factor,
        # ... остальные нутриенты
    }
```

### 7.3 Алгоритм формирования плана

```python
def generate_feeding_plan(
    pet: PetProfile,
    daily_calories: float,
    planned_activities: list[Activity],
    food_preferences: FoodPreferences
) -> FeedingPlan:
    """
    Генерирует план питания на 14 дней.
    """
    plan = FeedingPlan()
    
    for day in range(14):
        # 1. Базовые калории
        base_kcal = daily_calories
        
        # 2. Добавляем калории за активности этого дня
        day_activities = [a for a in planned_activities if a.day == day]
        activity_kcal = sum(
            calculate_activity_calories(a, pet.weight_kg) 
            for a in day_activities
        )
        
        # 3. Итоговая калорийность дня
        total_kcal = base_kcal + activity_kcal
        
        # 4. Валидация (не более +50% от базы)
        max_kcal = base_kcal * 1.5
        if total_kcal > max_kcal:
            total_kcal = max_kcal
            plan.warnings.append(f"День {day+1}: калорийность ограничена до +50%")
        
        # 5. Распределение по приемам пищи
        meals = distribute_to_meals(total_kcal, pet.species, pet.age_months)
        
        # 6. Расчет порций в граммах
        for meal in meals:
            meal.grams = meal.kcal / food_preferences.food_kcal_per_100g * 100
        
        plan.days.append(DayPlan(
            day=day,
            total_kcal=total_kcal,
            base_kcal=base_kcal,
            activity_kcal=activity_kcal,
            meals=meals
        ))
    
    return plan
```
