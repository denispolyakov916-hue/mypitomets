# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Единый Checkout для Фронтенда

## КОНТЕКСТ ПРОЕКТА

### Текущая Архитектура
**Платформа "Питомец+":** Веб-приложение для продажи товаров и курсов для животных.

**Текущее состояние:**
- Отдельные процессы оформления для товаров (`/api/shop/checkout/`) и курсов (`/api/courses/{id}/checkout/`)
- Корзина может содержать одновременно товары и курсы
- Пользователь вынужден оформлять покупки раздельно

**Проблема:**
- Разделённый UX для товаров и курсов
- Нет единого процесса оплаты
- Отсутствует резервирование товаров на время оформления

## ЦЕЛИ ПРОЕКТА

### Основная Цель
Создать единый процесс оформления заказа, позволяющий пользователю одновременно оплатить товары и курсы из корзины.

### Бизнес-Цели
1. **Улучшение конверсии:** Единый checkout снижает отказы от покупки
2. **Упрощение UX:** Один процесс вместо двух раздельных
3. **Защита от перепродаж:** Резервирование товаров на 10 минут
4. **Гибкость оплаты:** Единый платёж для всех типов покупок

## ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 1. Единый Checkout Процесс

#### 1.1 Вход в Checkout
- **Триггер:** Кнопка "Оформить заказ" в корзине
- **Условие активации:** Корзина содержит хотя бы один товар или курс
- **URL:** `/checkout` (новый маршрут)
- **API endpoint:** `GET /api/checkout/`

#### 1.2 Загрузка Данных
```
GET /api/checkout/
Response: {
  "products": {
    "items": [...],           // Товары из корзины
    "subtotal": 3500.00,      // Сумма товаров
    "delivery_options": [...] // Варианты доставки
  },
  "courses": {
    "items": [...],           // Курсы из корзины
    "subtotal": 5000.00       // Сумма курсов
  },
  "addresses": [...],         // Адреса пользователя
  "totals": {
    "products": 3500.00,
    "courses": 5000.00,
    "delivery": 0,           // Рассчитывается динамически
    "grand_total": 8500.00   // Общая сумма
  }
}
```

#### 1.3 Резервирование
- **Тайминг:** Автоматически при входе на страницу checkout
- **Продолжительность:** 10 минут
- **Что резервируется:**
  - Товары: уменьшение stock_count в БД
  - Курсы: блокировка мест на курс
- **Откат:** При истечении времени или ошибке

### 2. UI Компоненты

#### 2.1 ReservationTimer
**Расположение:** В верхней части страницы checkout
**Функционал:**
- Отсчёт времени: "⏰ Время на оплату: 09:45"
- Предупреждение за 1 минуту: красный цвет + текст "Поспешите!"
- Действие при истечении: редирект в корзину + сообщение об истечении

#### 2.2 ProductsSection (если есть товары)
**Содержимое:**
- Список товаров с изображениями, названиями, ценами
- Выбор типа доставки (стандартная/экспресс/самовывоз)
- Выбор адреса доставки (из списка адресов пользователя)
- Динамический расчёт стоимости доставки

#### 2.3 CoursesSection (если есть курсы)
**Содержимое:**
- Список курсов с названиями, ценами, привязкой к питомцам
- Чекбокс "Я принимаю условия использования курсов"
- Валидация: чекбокс обязателен для оформления

#### 2.4 SummarySection
**Расчёт сумм:**
```
Товары: 3500 ₽
Доставка: 300 ₽  (если выбрана)
Курсы: 5000 ₽
─────────────────
Итого: 8800 ₽
```

**Кнопка оплаты:** "Оплатить 8800 ₽"
- Отключена, если не приняты условия курсов
- Показывает loader во время отправки

### 3. Форма и Валидация

#### 3.1 Поля Формы
```javascript
const [formData, setFormData] = useState({
    // Для товаров (обязательны, если есть товары)
    delivery_type: 'standard',     // standard | express | pickup
    address_id: '',                // UUID адреса
    shipping_address: '',          // Альтернативный адрес текстом

    // Для курсов (обязательны, если есть курсы)

```

#### 3.2 Валидация
- **Если есть товары:** delivery_type и (address_id ИЛИ shipping_address) обязательны
- **Если есть курсы:** courses_disclaimer_accepted = true обязательно
- **Frontend валидация:** Предотвращает отправку формы с ошибками
- **Backend валидация:** Дублирующая проверка на сервере

### 4. API Интеграция

#### 4.1 Отправка Заказа
```
POST /api/checkout/
Body: {
  "delivery_type": "standard",
  "address_id": "uuid-address",
  "courses_disclaimer_accepted": true
}

Response: {
  "reservation_id": "uuid-reservation",
  "orders": {
    "products_order": {...},    // Созданный Order для товаров
    "courses": [...]            // Созданные UserCourse для курсов
  },
  "payment": {
    "id": "uuid-payment",
    "amount": 8800.00,
    "url": "https://payment.gateway/pay/uuid-payment"
  }
}
```

#### 4.2 Обработка Ошибок
- **Сетевая ошибка:** Показать toast "Ошибка сети, попробуйте ещё раз"
- **Валидация:** Выделить красным поля с ошибками
- **Истекло резервирование:** Редирект в корзину с сообщением
- **Недостаточно товаров:** Обновить корзину и показать предупреждение
