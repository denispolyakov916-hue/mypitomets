### 1.2 Ключевые изменения

| Было (wizard) | Стало (одна страница) |
|---------------|----------------------|
| 7 шагов по одному полю | Одна страница со всеми полями |
| Несколько кнопок "Далее" | Одна кнопка "Добавить питомца" |
| Экран "PetID создан!" | Сразу переход на Этап 2 |
| Раздельное сохранение | Одно сохранение → триггер → Этап 2 |

### 2.2 Порядок полей на форме

| # | Поле | Тип | Обязательное | Описание |
|---|------|-----|--------------|----------|
| 1 | **Фото** | Загрузка | Нет | Вверху формы, превью в кружке |
| 2 | **Имя** | Текст | Да | 1-100 символов |
| 3 | **Тип** | Выбор | Да | Собака / Кошка (кнопки) |
| 4 | **Пол** | Выбор | Да | Мальчик / Девочка (кнопки) |
| 5 | **Порода** | Автокомплит | Да | Поиск + популярные + "Дворняга" |
| 6 | **Возраст** | Дата/Число | Да | Точная дата ИЛИ примерный возраст |
| 7 | **Вес** | Число | Да | В кг, с подсказкой нормы |
| 8 | **Кастрация** | Выбор | Да | Да / Нет (кнопки) |

### 2.3 Валидация формы

```javascript
const validateForm = (data) => {
  const errors = {};
  
  if (!data.name?.trim()) errors.name = 'Введите имя питомца';
  if (!data.species) errors.species = 'Выберите тип питомца';
  if (!data.sex) errors.sex = 'Укажите пол';
  if (!data.breed_id && !data.is_mixed) errors.breed = 'Выберите породу';
  if (!data.date_of_birth && !data.age_months) errors.age = 'Укажите возраст';
  if (!data.weight_kg || data.weight_kg <= 0) errors.weight = 'Укажите вес';
  if (data.is_neutered === null) errors.neutered = 'Укажите кастрацию';
  
  return errors;
};
```

### 2.4 Действие при нажатии "Добавить питомца"

**1. Валидация:**
- Проверка всех обязательных полей
- Показ ошибок у невалидных полей

**2. Отправка на сервер:**

```javascript
// POST /api/pets/
const payload = {
  name: "Барон",
  species: "dog",
  breed_id: 123,           // null если дворняга
  date_of_birth: "2022-03-15",  // рассчитывается из возраста
  sex: "male",
  weight_kg: 28.5,
  is_neutered: true,
  photo: "base64..." | null
};
```

**3. Триггер автозаполнения (на сервере):**

```python
# Если breed_id != NULL:
size_category = breed.size              # "large"
coat_type = breed.coat_type             # "short"
ideal_weight_kg = breed.weight_range    # 29.5
activity_level = adjust_by_age(breed.activity_level, age_months)  # "high"

# Если breed_id == NULL (дворняга):
size_category = calculate_by_weight_age(species, weight_kg, age_months)
coat_type = NULL                        # Пользователь укажет на Этапе 2
ideal_weight_kg = NULL
activity_level = "moderate"

# Всегда:
body_condition_score = calculate_bcs(weight_kg, ideal_weight_kg)
```

**4. Response от сервера:**

```json
{
  "id": "uuid-pet-id",
  "name": "Барон",
  "species": "dog",
  "breed": {"id": 123, "name": "Лабрадор ретривер"},
  "date_of_birth": "2022-03-15",
  "sex": "male",
  "weight_kg": 28.5,
  "is_neutered": true,
  
  "size_category": "large",
  "coat_type": "short",
  "ideal_weight_kg": 29.5,
  "activity_level": "high",
  "body_condition_score": 5,
  
  "age_months": 46,
  "age_category": "adult",
  "profile_completeness": 35
}
```

**5. Переход на Этап 2:**
- БЕЗ промежуточного экрана "PetID создан!"
- Сразу открываем страницу расширенного профиля с автозаполненными данными

## 5. Подбор корма

### 5.1 Доступность подбора корма

**Подбор корма доступен СРАЗУ после Этапа 1** (базовый профиль).

Минимальные данные для расчёта:
- `species` — вид (dog/cat)
- `weight_kg` — вес
- `date_of_birth` — возраст
- `is_neutered` — кастрация

Все остальные данные:
- Автозаполняются из породы (если указана)
- Используются defaults (если дворняга)
- Уточняются из Этапа 2 (если заполнен)

### 5.2 Предупреждение о неполном профиле

Если профиль заполнен < 50%:

```
┌─────────────────────────────────────────────────────────────────────┐
│  ℹ️ Профиль Барона заполнен на 35%                                  │
│                                                                     │
│  Для более точного подбора рекомендуем:                            │
│  • Указать текущий корм                                            │
│  • Добавить информацию о здоровье                                  │
│  • Указать активности                                              │
│                                                                     │
│  [Заполнить сейчас]    [Продолжить с текущими данными]             │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.3 Ситуация "Нет подходящих кормов"

**Такой ситуации НЕ БУДЕТ**, потому что:

1. **В магазине всегда есть базовые корма** без аллергенов
2. **Фильтрация постепенная:**
   - Сначала по виду (dog/cat)
   - Потом по возрасту (puppy/adult/senior)
   - Потом по размеру (если собака)
   - Потом исключаем аллергены
3. **Если нет идеальных → показываем "хорошо подходящие"**
4. **Гипоаллергенные корма** — всегда есть как fallback

**Единственный крайний случай:** Пользователь вручную указал аллергию на ВСЕ основные белки (курица + говядина + рыба + ягнёнок + свинина). Но это маловероятно.

### 5.4 Влияние текущего корма

Поле `current_food` в PetID влияет **только на уведомление о смене типа**:

```
┌─────────────────────────────────────────────────────────────────────┐
│  ⚠️ Вы переходите с сухого корма на влажный                        │
│                                                                     │
│  Резкая смена типа питания может вызвать расстройство ЖКТ.         │
│  Рекомендуем постепенный переход в течение 7-10 дней.              │
│                                                                     │
│  [Понятно, продолжить]                                             │
└─────────────────────────────────────────────────────────────────────┘
```

Никакого влияния на приоритет брендов или продуктов в рекомендациях.
