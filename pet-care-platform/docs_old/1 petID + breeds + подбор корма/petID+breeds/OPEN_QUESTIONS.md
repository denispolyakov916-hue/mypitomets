## ✅ Все вопросы решены

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | **Калорийность кормов** | Использовать реальные данные из БД (`product.kcal_per_100g`). Добавить примерные значения для каждого корма, потом обновить на реальные. |
| 2 | **Максимум мелких упаковок** | Оба варианта: показывать как есть (15 паучей) + предлагать коробку побольше. Пользователь сам выбирает. |
| 3 | **"Часто покупают вместе"** | Автоматика на основе данных + приоритеты по продаже определённых производителей. |
| 4 | **Автоматический расчёт полей Product** | Всё добавляется для каждого товара отдельно. Примерные значения сейчас, реальные потом. |
| 5 | **Распределение калорий** | 60/30/10 для мульти, 90/10 для моно — да, если соответствует действительности. |
| 6 | **Конфликты терапевтических кормов** | В финальной подборке НЕ должно быть конфликтов. Пользователь выбирает только из совместимых кормов. |
| 7 | **Минимум заполненности PetID** | Нет минимума. Подбор доступен сразу после Этапа 1 (базовый профиль). |
| 8 | **Расчёт "хватит на X дней"** | Да, учитывать активные дни в расчёте. |
| 9 | **"Нет подходящих кормов"** | Такой ситуации НЕ будет: всегда есть гипоаллергенные и базовые корма. |
| 10 | **Влияние текущего корма** | Только предупреждение о смене типа (вредно для ЖКТ). Никакого влияния на рекомендации. |

## Детали решений

### 1. Калорийность кормов

```python
# Каждый корм в БД ДОЛЖЕН иметь:
class Product(models.Model):
    kcal_per_100g = models.DecimalField(...)  # Обязательное
    nutrition_facts = models.JSONField(...)   # {protein, fat, fiber, ash, moisture}
    
# НЕ используем дефолтные значения по категориям!
# Добавляем примерные данные для каждого корма сейчас,
# обновляем на реальные данные от поставщика позже.
```

### 2. Упаковки — оба варианта

Пользователю показываем:
- Вариант A: "30 паучей × 85г = 2 550г"
- Вариант B: "Коробка 24 шт + 6 отдельно" (если выгоднее)

Пользователь сам выбирает удобный вариант.

### 3. "Часто покупают вместе"

Алгоритм:
1. Анализ заказов пользователей с похожими питомцами
2. Ручные приоритеты брендов/производителей в админке
3. Комбинация с весами: 70% автоматика + 30% ручные настройки

### 6. Совместимость в подборке

```python
# В финальную подборку попадают ТОЛЬКО совместимые корма
def get_compatible_products(pet, base_product):
    """
    Если пользователь выбрал сухой корм X,
    показываем влажные корма, которые СОВМЕСТИМЫ с X.
    
    Несовместимые варианты просто не отображаются.
    """
    compatible_group = get_compatibility_group(base_product)
    
    return Product.objects.filter(
        compatibility_groups__contains=[compatible_group]
    )
```

### 9. Почему "нет кормов" не случится

1. **В магазине всегда есть:**
   - Гипоаллергенные корма (моно-протеин)
   - Базовые корма без экзотических ингредиентов
   
2. **Фильтрация постепенная:**
   - Вид → Возраст → Размер → Аллергены
   - Если на шаге N результатов мало — ослабляем критерии
   
3. **Показываем "хорошо подходящие":**
   - Если нет "идеальных" (score >= 90)
   - Показываем "хорошо подходящие" (score >= 70)

### 10. Текущий корм — только уведомление

```javascript
// Проверка при подборе
if (pet.current_food?.diet_type && newPlan.feeding_type !== pet.current_food.diet_type) {
  showWarning({
    title: "Смена типа питания",
    message: "Резкая смена типа питания может вызвать расстройство ЖКТ. " +
             "Рекомендуем постепенный переход в течение 7-10 дней.",
    actions: ["Понятно, продолжить", "Отмена"]
  });
}
```

## Изменения в PetID

### Форма создания — одна страница

**Порядок полей:**
1. Фото (вверху формы)
2. Имя
3. Тип (собака/кошка)
4. Пол
5. Порода
6. Возраст
7. Вес
8. Кастрация

**После нажатия "Добавить питомца":**
1. Сохранение в БД
2. Триггер автозаполнения
3. **Сразу переход на Этап 2** (без экрана "PetID создан!")

### Этап 2 — расширенный профиль

Пользователь видит:
- Автозаполненные данные с меткой "✨ Авто"
- Блоки для заполнения (здоровье, питание и т.д.)
- Кнопка "Сохранить изменения"

### Главная страница

Рядом с карточкой питомца:
- 📊 Анализ питомца
- 🍽️ Подбор корма

В хедере сайта — отдельная кнопка "Подбор корма".
