# План реализации перестановки страниц и модулей в конструкторе курсов

## Введение

Реализуем функционал конструктора курсов — инструмента для создания и редактирования образовательных курсов. Курс состоит из модулей, модуль — из страниц (уроков). Страницы могут быть привязаны к модулю или находиться «без модуля» (orphan). Нужно дать возможность менять порядок страниц внутри модуля, порядок страниц в блоке «Без модуля» и порядок модулей в курсе.

**Расположение интерфейса:**
- **Слева** — дерево структуры (модули и страницы), кнопки «+ Модуль», «+ Страница», удаление.
- **По центру** — холст редактирования текущей страницы (блоки контента).
- **Справа** — панель «Свойства» выбранного элемента (страница, модуль или блок).

Backend API для перестановки уже есть. Нужно подключить frontend.

---

## Этап 1. Стрелки в панели свойств (справа)

**Где:** правая панель «Свойства» (`PropertiesPanel.jsx`).

**Что делаем:**
1. Когда выбран **модуль** — показываем кнопки «Вверх» и «Вниз» для изменения его позиции среди модулей курса.
2. Когда выбран **страница** — показываем кнопки «Вверх» и «Вниз» для изменения её позиции внутри модуля или в блоке «Без модуля».

**Проверка:** заказчик проверяет реализацию.

**Исправления:** при необходимости вносим правки по результатам проверки.

---

## Этап 2. Drag & Drop (после одобрения Этапа 1)

**Где:** дерево структуры слева (`ModuleTree` в `CanvasArea.jsx` или `StructureTree.jsx`).

**Что делаем:** добавляем возможность перетаскивать страницы и модули в дереве для изменения порядка.

**Проверка:** заказчик проверяет реализацию.

**Исправления:** при необходимости вносим правки.

---

## Порядок работ

1. Реализовать **Этап 1** (стрелки в панели свойств).
2. Дождаться проверки и одобрения заказчиком.
3. Только после одобрения переходить к **Этапу 2** (DnD).
