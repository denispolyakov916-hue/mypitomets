# Ролевая модель и превью курсов

## 1. Роли пользователей

В системе три роли. Источник правды — поле `role` в модели `User`.
Django-флаги `is_staff` / `is_superuser` синхронизируются автоматически при сохранении.

| Роль               | `role`           | `is_staff` | `is_superuser` | Доступ к админ-панели      |
|--------------------|------------------|------------|----------------|----------------------------|
| Пользователь       | `user`           | `False`    | `False`        | Нет                        |
| Создатель курсов   | `course_creator` | `False`    | `False`        | Да (только свои курсы)     |
| Администратор      | `admin`          | `True`     | `True`         | Да (полный + назначение ролей) |

### Синхронизация в `User.save()`

```python
def save(self, *args, **kwargs):
    if self.role == 'admin':
        self.is_staff = True
        self.is_superuser = True
    else:
        self.is_staff = False
        self.is_superuser = False
    super().save(*args, **kwargs)
```

### Назначение ролей

- Роли назначает **только Администратор** через форму редактирования пользователя в админ-панели.
- Создатель курсов и обычный пользователь **не могут** менять роли.

## 2. Связь курса с автором

Новое поле в модели `Course`:

```python
author = models.ForeignKey(
    'users.User',
    on_delete=models.SET_NULL,
    null=True, blank=True,
    related_name='authored_courses',
    verbose_name='Автор курса'
)
```

- При создании курса `author` устанавливается автоматически на текущего пользователя.
- Администратор может переназначить автора.

## 3. Матрица доступа к курсам

| Действие                    | Администратор | Создатель курсов   | Пользователь |
|-----------------------------|---------------|---------------------|--------------|
| Видит курсы в админке       | Все           | Только свои         | —            |
| Создаёт курс                | Да            | Да                  | Нет          |
| Редактирует курс            | Все           | Только свои         | Нет          |
| Предпросмотр контента курса | Все           | Только свои         | Нет          |
| Публикует курс              | Да            | **Нет**             | Нет          |
| Снимает с публикации        | Да            | **Нет**             | Нет          |
| Удаляет курс                | Да            | **Нет**             | Нет          |
| Назначает роли              | Да            | Нет                 | Нет          |
| Проходит купленный курс     | Да            | Да                  | Да           |

## 4. Жизненный цикл курса

```
Создатель/Админ создаёт курс
        │
        ▼
    [draft]  ◄──── Создатель редактирует, сохраняет, предпросматривает
        │
        │  Администратор публикует
        ▼
  [published]  ───► Доступен пользователям в каталоге
        │
        │  Администратор снимает с публикации
        ▼
   [archived]
```

- Создатель курса работает **только** со статусом `draft`.
- Кнопки «Опубликовать» / «Снять с публикации» / «Удалить» ему **не показываются**.

## 5. Предпросмотр курса

### Проблема (до)

Кнопка «Предпросмотр» открывала `/courses/{id}` — публичную карточку курса,
где нужно купить курс для доступа к контенту.

### Решение (после)

Кнопка «Предпросмотр» открывает `/training/courses/{id}/learn` — страницу обучения.
Доступ к контенту разрешается через централизованную проверку `has_course_access()`.

### Централизованная функция проверки доступа

```python
def has_course_access(user, course):
    """Может ли пользователь просматривать контент курса."""
    if user.is_staff or user.is_superuser:
        return True
    if course.author_id and course.author_id == user.id:
        return True
    return UserCourse.objects.filter(user=user, course=course).exists()
```

Применяется во всех вью, которые отдают контент курса:
- `CoursePageLearningView` — страницы курса
- `CourseStructureView` — структура курса
- `VideoPlaybackUrlView` — воспроизведение видео
- Комментарии к урокам
