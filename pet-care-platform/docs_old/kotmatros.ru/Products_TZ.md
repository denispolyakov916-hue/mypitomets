# Products_TZ

Цель: структурировать данные товаров в нормализованном виде, чтобы:
- одинаково быстро заполнять карточку товара в каталоге и на детальной странице;
- иметь полный набор параметров по каждому SKU/варианту;
- связать товары в группы (SKU/варианты) и поддержать фильтры/поиск (в том числе по вкусу);
- легко расширять категории без переписывания базы.

## 1. Подход и принципы
1) **Core + Category**: единое ядро товара + отдельные таблицы для категорий.
2) **SKU/Variant**: все торговые позиции (вес, упаковка, вкус) живут в SKU.
3) **Нормализация**: справочники (бренд, производитель, вкус, ингредиент, витамин).
4) **Гибкая атрибутика**: атрибуты по категориям, но без EAV-помойки.
5) **Поиск и витрина**: индексируемый слой/представление для быстрых витрин.

Практика современных сервисов: **гибрид** — нормализованные таблицы + денормализованный индекс (search_view/materialized view), чтобы быстро собирать карточки и фильтры.

## 3. Базовая модель данных (общая для всех товаров)
Минимальный набор сущностей:
- **Product**: карточка товара как сущность (название, бренд, категория).
- **SKU/Variant**: конкретная торговая позиция (вкус/вес/упаковка).
- **Category**: дерево категорий и подкатегорий.
- **Brand** / **Manufacturer**: бренды и производители.
- **Media**: изображения/видео.
- **Pricing**: цена, скидка, валюта.
- **Stock**: наличие, остаток, статус.
- **Attributes**: стандартизированные атрибуты для фильтров.

## 4. SKU/варианты и связи товаров
Связи и правила:
- один **Product** имеет несколько **SKU**;
- SKU связаны в **variant_group** (например, тот же корм 400/800/2000 г);
- SKU может иметь **flavor** и **package** как отдельные справочники;
- поиск по вкусу работает по SKU и возвращает карточки Product.

## 5. Общие поля для отображения (каталог и карточка)
### 5.1 Для каталога (быстрый вывод)
- product_id, sku_id
- название, бренд, категория, тип животного (кошка/собака)
- цена/скидка/валюта
- изображение превью
- вес/объем упаковки
- вкус (если применимо)
- возраст/размер (если применимо)
- наличие/статус

### 5.2 Для детальной карточки
- все из каталога
- описание, состав, рекомендации, условия хранения
- характеристики (в формате ключ-значение)
- медиа-галерея
- связанные SKU (варианты)

## 6. Категорийные таблицы (детальные данные)

### 6.1 Корма (Food)
Цель: хранить состав нутриентов, витаминов, калорийность, БЖУ, вес нетто.
- **food_profile** (на уровне SKU)
  - energy_kcal_per_100g
  - protein_g_per_100g
  - fat_g_per_100g
  - carbs_g_per_100g
  - fiber_g_per_100g
  - ash_g_per_100g
  - moisture_percent
  - net_weight_g
  - daily_portion_guidelines (json)
- **food_ingredients**
  - ingredient_id, percent, order
- **food_nutrients**
  - nutrient_id, value, unit
- **food_vitamins**
  - vitamin_id, value, unit
- **food_minerals**
  - mineral_id, value, unit
- **food_constraints**
  - life_stage, breed_size, activity_level
  - medical_use, allergies, special_diet

### 6.2 Лакомства (Treats)
- вкус, назначение (обучение/стоматология), форма
- калорийность (если есть), состав
- ограничения по возрасту/размеру

### 6.3 Ветаптека (Vet Pharmacy)
- действующее вещество, форма выпуска, дозировка
- назначение, противопоказания
- инструкция, курс применения

### 6.4 Витамины и добавки
- состав (витамины/минералы), дозировка
- форма (таблетки/капсулы/порошок)
- курс, противопоказания

### 6.5 Наполнители
- тип (силикагель/комкующийся/древесный)
- объем/вес, аромат
- впитываемость, утилизация

### 6.6 Миски и поилки
- материал, объем, особенности (антискользящая/авто)
- совместимость (клетки/переноски)

### 6.7 Игрушки
- материал, размер, тип (интерактив/жевалка)
- безопасность, возраст

### 6.8 Амуниция
- тип, размер, материал, длина/ширина
- регулируемость, нагрузка

### 6.9 Средства по уходу и груминг
- тип средства/инструмента
- состав (если косметика), назначение
- объем/вес, меры предосторожности

### 6.10 Когтеточки
- материал, тип установки, размер

### 6.11 Транспортировка и содержание
- тип (переноска/лежанка/клетка)
- материал, размер/габариты
- комплектация, нагрузка

### 6.12 Одежда
- размер, материал, сезон
- пол/порода/размер животного

### 6.13 Туалеты и принадлежности
- тип, размер, совместимость с наполнителями
- расходники

### 6.14 Контроль поведения
- тип (ошейники, спреи, кликеры)
- назначение, ограничения

## 7. Поиск и фильтры (вкус, состав и др.)
Рекомендованный слой:
- **search_index**: объединяет Product + SKU + ключевые атрибуты.
- Индексация полей: вкус, ингредиенты, бренд, возраст, размер, назначение.
- Поиск по вкусу: фильтрация SKU → возврат карточек Product.

## 8. Дальнейшие шаги заполнения
1) Утвердить общий набор **core-полей** (раздел 5).
2) Утвердить список **категорийных таблиц** (раздел 6).
3) Для каждой категории выписать **точные поля** и типы данных.
4) Определить, какие поля нужны в **быстрый каталог** и в **детальную страницу**.
5) Согласовать **поиск/фильтры** и словари (вкус, ингредиенты, витамины).
