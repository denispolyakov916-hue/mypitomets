## Общая архитектура взаимодействия

**Разделение ответственности:**
- **pitometsplus.ru** - оформление заказов, приём оплаты
- **kotmatros.ru** - доставка, логистика, обработка статусов заказов

**Ключевые принципы:**
- ID товаров одинаковые на обоих сервисах (источник - kotmatros.ru)
- Односторонняя синхронизация товаров (от kotmatros.ru к pitometsplus.ru) при оформлении заказов
- Бронирование сначала на kotmatros.ru, затем локально на pitometsplus.ru
- Webhook-механизм для уведомлений о статусах заказов

**Ключевые таймауты:**
- **Бронь на kotmatros.ru:** 1 час (общее время на оформление и оплату)
- **Таймер оплаты на pitometsplus.ru:** 15 минут (начинается при переходе к оплате)
- **Retry при бронировании:** 3 попытки с интервалами 1с → 5с → 15с
- **Retry при создании заказа:** каждые 5 секунд в течение 2 минут (после оплаты)

## Принципы надёжности

### Идемпотентность операций

Все критичные операции используют механизм идемпотентности через заголовок `X-Idempotency-Key: {uuid}`.

При повторном запросе с тем же ключом сервер возвращает сохранённый результат без повторного выполнения.

**Применяется к:**
- Создание брони
- Создание заказа
- Возврат товара

### Обработка ошибок и повторы

| Параметр | Значение |
|----------|----------|
| Количество попыток | 3 |
| Интервалы | 1 сек → 5 сек → 15 сек |
| Повторять при | HTTP 5xx, таймаут, сетевые ошибки |
| НЕ повторять при | HTTP 4xx, HTTP 2xx |

## API эндпоинты

### Эндпоинты pitometsplus.ru (принимает от kotmatros.ru)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/kotmatros/products/` | Создание/обновление товара (upsert) |
| PATCH | `/api/kotmatros/products/{id}/` | Частичное обновление товара |
| DELETE | `/api/kotmatros/products/{id}/` | Мягкое удаление товара |
| POST | `/api/kotmatros/products/sync/` | Массовая синхронизация (delta за 3 часа) |
| POST | `/api/kotmatros/webhooks/orders/status/` | Webhook: обновление статуса заказа |
| POST | `/api/kotmatros/webhooks/orders/refund/` | Webhook: результат возврата товара |
| POST | `/api/kotmatros/webhooks/reservations/expired/` | Webhook: истечение брони |

### Эндпоинты kotmatros.ru (принимает от pitometsplus.ru)

| Метод | Эндпоинт | Описание | Idempotency |
|-------|----------|----------|-------------|
| POST | `/api/pitometsplus/reservations/` | Создание брони | ✅ |
| DELETE | `/api/pitometsplus/reservations/{id}/` | Освобождение брони | - |
| POST | `/api/pitometsplus/orders/` | Создание заказа | ✅ |
| DELETE | `/api/pitometsplus/orders/{id}/` | Отмена заказа | - |
| POST | `/api/pitometsplus/orders/{id}/refund/` | Возврат товара | ✅ |
| GET | `/api/pitometsplus/orders/{id}/status/` | Проверка статуса | - |

## Use Cases

### UC-01: Оформление и оплата заказа

| Поле | Описание |
|------|----------|
| **Название** | Оформление и оплата заказа на pitometsplus.ru |
| **Актёры** | Покупатель, pitometsplus.ru, kotmatros.ru, Платёжная система |
| **Предусловия** | 1. Покупатель авторизован на pitometsplus.ru<br>2. В корзине есть минимум один товар<br>3. Товары доступны на складе kotmatros.ru<br>4. Платёжная система доступна<br>5. kotmatros.ru доступен |
| **Триггер** | Покупатель нажимает кнопку "Оформить заказ" в корзине |
| **Результат** | 1. Заказ создан на pitometsplus.ru и kotmatros.ru<br>2. Товары списаны со склада<br>3. Средства списаны с карты покупателя<br>4. Покупатель получил подтверждение заказа |

#### Основной сценарий

| Шаг | Актёр | Действие |
|-----|-------|----------|
| 1 | Покупатель | Нажимает "Оформить заказ" в корзине |
| 2 | pitometsplus.ru | Генерирует `reservation_id` |
| 3 | pitometsplus.ru → kotmatros.ru | `POST /api/pitometsplus/reservations/` с товарами |
| 4 | kotmatros.ru | Проверяет наличие, бронирует товары на 1 час |
| 5 | kotmatros.ru → pitometsplus.ru | Возвращает подтверждение брони |
| 6 | pitometsplus.ru | Создаёт локальную бронь, показывает форму оформления |
| 7 | Покупатель | Заполняет данные доставки, выбирает способ оплаты |
| 8 | Покупатель | Нажимает "Оплатить" |
| 9 | pitometsplus.ru | Запускает таймер 15 минут, перенаправляет на оплату |
| 10 | Покупатель | Вводит данные карты, подтверждает оплату |
| 11 | Платёжная система → pitometsplus.ru | Возвращает подтверждение оплаты |
| 12 | pitometsplus.ru | Генерирует `order_id` |
| 13 | pitometsplus.ru → kotmatros.ru | `POST /api/pitometsplus/orders/` с данными заказа |
| 14 | kotmatros.ru | Конвертирует бронь в заказ, списывает товары |
| 15 | kotmatros.ru → pitometsplus.ru | Возвращает `external_order_id` |
| 16 | pitometsplus.ru | Создаёт заказ локально, обновляет остатки |
| 17 | pitometsplus.ru | Показывает страницу "Заказ успешно оформлен" |
| 18 | pitometsplus.ru | Отправляет email-подтверждение покупателю |

### UC-04: Синхронизация каталога товаров

| Поле | Описание |
|------|----------|
| **Название** | Синхронизация каталога товаров с kotmatros.ru |
| **Актёры** | kotmatros.ru, pitometsplus.ru |
| **Предусловия** | 1. Оба сервиса доступны<br>2. ID товаров одинаковые на обоих сервисах |
| **Триггер** | Изменение товара на kotmatros.ru / Периодическая синхронизация (каждые 3 часа) |
| **Результат** | Каталог pitometsplus.ru актуален и соответствует kotmatros.ru |

#### Основной сценарий: Добавление нового товара

| Шаг | Актёр | Действие |
|-----|-------|----------|
| 1 | kotmatros.ru | Добавляет новый товар в каталог |
| 2 | kotmatros.ru → pitometsplus.ru | `POST /api/kotmatros/products/` с данными товара |
| 3 | pitometsplus.ru | Создаёт товар в своём каталоге |
| 4 | pitometsplus.ru → kotmatros.ru | Возвращает HTTP 201 Created |

#### Основной сценарий: Обновление товара

| Шаг | Актёр | Действие |
|-----|-------|----------|
| 1 | kotmatros.ru | Изменяет данные товара (цена, количество, описание) |
| 2 | kotmatros.ru → pitometsplus.ru | `PATCH /api/kotmatros/products/{id}/` с изменёнными полями |
| 3 | pitometsplus.ru | Обновляет товар (или создаёт, если не существует) |
| 4 | pitometsplus.ru → kotmatros.ru | Возвращает HTTP 200 OK |

#### Основной сценарий: Удаление товара

| Шаг | Актёр | Действие |
|-----|-------|----------|
| 1 | kotmatros.ru | Снимает товар с продажи |
| 2 | kotmatros.ru → pitometsplus.ru | `DELETE /api/kotmatros/products/{id}/` |
| 3 | pitometsplus.ru | Выполняет мягкое удаление:<br>• Скрывает товар из каталога<br>• Обнуляет количество<br>• Сохраняет для существующих заказов |
| 4 | pitometsplus.ru → kotmatros.ru | Возвращает HTTP 200 OK |

#### Основной сценарий: Периодическая синхронизация (delta)

| Шаг | Актёр | Действие |
|-----|-------|----------|
| 1 | kotmatros.ru | Таймер периодической синхронизации |
| 2 | kotmatros.ru → pitometsplus.ru | `POST /api/kotmatros/products/sync/` с товарами, изменёнными за 3 часа |
| 3 | pitometsplus.ru | Применяет все изменения (создание/обновление/удаление) |
| 4 | pitometsplus.ru → kotmatros.ru | Возвращает HTTP 200 OK |

### UC-05: Обновление статуса заказа

| Поле | Описание |
|------|----------|
| **Название** | Обновление статуса заказа через webhook |
| **Актёры** | kotmatros.ru, pitometsplus.ru, Покупатель |
| **Предусловия** | 1. Заказ существует на обоих сервисах<br>2. pitometsplus.ru доступен для webhook |
| **Триггер** | Изменение статуса заказа на kotmatros.ru |
| **Результат** | 1. Статус заказа синхронизирован<br>2. Покупатель уведомлён |

#### Основной сценарий

| Шаг | Актёр | Действие |
|-----|-------|----------|
| 1 | kotmatros.ru | Изменяет статус заказа (processing → shipped → delivered) |
| 2 | kotmatros.ru → pitometsplus.ru | `POST /api/kotmatros/webhooks/orders/status/` с новым статусом |
| 3 | pitometsplus.ru | Обновляет статус заказа локально |
| 4 | pitometsplus.ru → kotmatros.ru | Возвращает HTTP 200 OK |
| 5 | pitometsplus.ru | Уведомляет покупателя (email/push) |
| 6 | pitometsplus.ru | Если статус "delivered" → предлагает оставить отзыв |

#### Статусы заказа

| Статус | Описание | Действия pitometsplus.ru |
|--------|----------|--------------------------|
| `processing` | В обработке / сборка | Уведомление покупателя |
| `shipped` | Отправлен / передан в доставку | Уведомление с трекингом |
| `delivered` | Доставлен | Уведомление, запрос отзыва |
| `problem` | Проблема с доставкой | Уведомление, предложение решений |

## Управление количеством товаров

### На kotmatros.ru (источник акутальных данных):
- Управляет реальными остатками товаров на складе
- При бронировании блокирует товары временно (1 час)
- При подтверждении заказа списывает товары со склада
- При возвратах/отменах возвращает товары на склад

### На pitometsplus.ru (отображение):
- Отображает остатки из синхронизации с kotmatros.ru
- Локально бронирует товары после подтверждения от kotmatros.ru
- Обновляет отображаемые остатки при получении webhook'ов
- Не управляет реальными остатками — только UI и локальная БД
