# 🚀 Запуск проекта Питомец+

## 📋 Быстрый старт

### Вариант 1: Запуск всего проекта сразу
```bash
./start_all.sh
```

### Вариант 2: Запуск по отдельности

#### Бэкенд (Django)
```bash
cd backend
./start_backend.sh
```

#### Фронтенд (React + Vite)
```bash
cd frontend
./start_frontend.sh
```

## 🌐 Доступ к приложению

После успешного запуска:

- **Главная страница**: http://localhost:5173
- **API бэкенда**: http://localhost:8000/api/
- **Админ-панель**: http://localhost:8000/admin/

## 📁 Структура скриптов

### `start_all.sh` - Запуск всего проекта
- Автоматически запускает бэкенд и фронтенд параллельно
- Показывает PID процессов для контроля
- Корректно останавливает все сервисы при завершении

### `backend/start_backend.sh` - Запуск бэкенда
Проверяет и выполняет:
- ✅ Наличие Python 3.10+
- ✅ Создание виртуального окружения
- ✅ Установка зависимостей из requirements.txt
- ✅ Наличие и создание .env файла
- ✅ Подключение к PostgreSQL базе данных
- ✅ Применение Django миграций
- ✅ Запуск сервера на порту 8000

### `frontend/start_frontend.sh` - Запуск фронтенда
Проверяет и выполняет:
- ✅ Наличие Node.js 18+
- ✅ Установка npm зависимостей
- ✅ Создание .env файла с настройками
- ✅ Запуск Vite dev-сервера на порту 5173

## 🔧 Системные требования

### Для бэкенда:
- Python 3.10+
- PostgreSQL 12+
- pip

### Для фронтенда:
- Node.js 18+
- npm

## ⚙️ Настройка базы данных

При первом запуске может потребоваться настройка PostgreSQL:

```bash
cd backend
./setup_db.sh
```

## 🔑 Переменные окружения

### Бэкенд (.env)
```bash
DB_NAME=pitomets_db
DB_USER=pitomets
DB_PASSWORD=pitomets_password
DB_HOST=localhost
DB_PORT=5432
DEBUG=True
DJANGO_SECRET_KEY=your-secret-key
CLIENT_URL=http://localhost:5173
API_URL=http://localhost:8000
```

### Фронтенд (.env)
```bash
VITE_API_URL=http://localhost:8000/api
```

## 🛠️ Устранение неполадок

### Бэкенд не запускается:
1. Проверьте что PostgreSQL запущен
2. Проверьте настройки в `backend/.env`
3. Попробуйте запустить `./backend/setup_db.sh`

### Фронтенд не запускается:
1. Проверьте версию Node.js: `node --version`
2. Удалите `node_modules` и запустите заново
3. Проверьте настройки в `frontend/.env`

### Порт уже используется:
- Бэкенд: порт 8000
- Фронтенд: порт 5173

Используйте `lsof -i :PORT` для проверки.

## 📝 Логи

Логи бэкенда выводятся в терминал при запуске.
Для фронтенда логи доступны в браузере (F12 → Console).
