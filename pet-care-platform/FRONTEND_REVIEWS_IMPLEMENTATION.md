# Реализация системы рейтингов и отзывов на фронтенде

## Описание

Реализована полная система рейтингов и отзывов для товаров и курсов с современным UX/UI дизайном в стиле e-commerce. Система позволяет пользователям просматривать рейтинги, оставлять отзывы только после покупки товара или курса, и управлять своими отзывами.

---

## Созданные компоненты

### 1. Rating.jsx
**Путь:** `src/components/Rating.jsx`

**Описание:** Компонент отображения рейтинга в виде 5 звезд.

**Функциональность:**
- Отображение рейтинга от 0 до 5 звезд
- Интерактивный выбор рейтинга (для форм)
- Только просмотр (readonly режим)
- Отображение среднего рейтинга с количеством отзывов
- Три размера: `sm`, `md`, `lg`
- Плавные анимации при наведении

**Использование:**
```jsx
<Rating
  rating={4.5}
  reviewsCount={25}
  readonly={true}
  size="md"
/>
```

**Props:**
- `rating` (number, 0-5) - Текущий рейтинг
- `reviewsCount` (number, опционально) - Количество отзывов
- `readonly` (boolean, default: true) - Только для просмотра
- `onChange` (function, опционально) - Обработчик изменения рейтинга
- `size` (string, 'sm'|'md'|'lg') - Размер звезд

### 2. ReviewForm.jsx
**Путь:** `src/components/ReviewForm.jsx`

**Описание:** Форма для создания и редактирования отзывов.

**Функциональность:**
- Выбор рейтинга (1-5 звезд)
- Текстовое поле для комментария (10-2000 символов)
- Валидация формы
- Счетчик символов
- Поддержка редактирования существующего отзыва
- Проверка права на отзыв (только для купивших)

**Использование:**
```jsx
<ReviewForm
  isPurchased={true}
  onSubmit={handleSubmit}
  isLoading={false}
  existingReview={null}
/>
```

**Props:**
- `isPurchased` (boolean) - Приобретен ли товар/курс
- `onSubmit` (function) - Обработчик отправки формы
- `isLoading` (boolean) - Состояние загрузки
- `existingReview` (object, опционально) - Существующий отзыв для редактирования

**Валидация:**
- Рейтинг обязателен (1-5)
- Комментарий: минимум 10 символов, максимум 2000 символов

### 4. ReviewsSection.jsx
**Путь:** `src/components/ReviewsSection.jsx`

**Описание:** Комплексный компонент секции отзывов, объединяющий форму и список.

**Функциональность:**
- Общая статистика рейтинга (средний рейтинг, количество отзывов)

**Особенности:**
- Автоматически загружает отзывы при монтировании
- Проверяет возможность оставить отзыв через API
- Показывает существующий отзыв пользователя для редактирования
- Обновляет список после создания/обновления отзыва

## API функции

### Файл: `src/api/reviews.js`

**Созданные функции:**

1. **getProductReviews(productId, params)**
   - Получение отзывов товара с пагинацией
   - `GET /api/reviews/products/{productId}/`

2. **getCourseReviews(courseId, params)**
   - Получение отзывов курса с пагинацией
   - `GET /api/reviews/courses/{courseId}/`

3. **createProductReview(productId, reviewData)**
   - Создание отзыва на товар
   - `POST /api/reviews/products/{productId}/`

4. **createCourseReview(courseId, reviewData)**
   - Создание отзыва на курс
   - `POST /api/reviews/courses/{courseId}/`

5. **updateProductReview(productId, reviewId, reviewData)**
   - Обновление отзыва на товар
   - `PUT /api/reviews/products/{productId}/reviews/{reviewId}/`

6. **updateCourseReview(courseId, reviewId, reviewData)**
   - Обновление отзыва на курс
   - `PUT /api/reviews/courses/{courseId}/reviews/{reviewId}/`

7. **deleteReview(type, itemId, reviewId)**
   - Удаление отзыва
   - `DELETE /api/reviews/{type}/{itemId}/reviews/{reviewId}/`

8. **checkReviewEligibility(type, itemId)**
   - Проверка возможности оставить отзыв
   - `GET /api/reviews/{type}/{itemId}/eligibility/`

## Логика работы

### 1. Проверка возможности оставить отзыв

1. При загрузке секции отзывов вызывается `checkReviewEligibility()`
2. API возвращает информацию о возможности оставить отзыв
3. Форма отзыва показывается только если `can_review === true`
4. Если отзыв уже существует, форма показывается в режиме редактирования

### 2. Отображение рейтинга

1. Рейтинг отображается на карточках и страницах деталей
2. Если рейтинг отсутствует (0 или null), компонент не отображается
3. Средний рейтинг округляется до 1 знака после запятой
4. Количество отзывов отображается в скобках

### 3. Создание/обновление отзыва

1. Пользователь выбирает рейтинг и вводит комментарий
2. Форма валидируется перед отправкой
3. При успешной отправке список отзывов обновляется
4. Показывается уведомление об успехе
5. Если отзыв уже существует, он обновляется вместо создания нового

### 4. Распределение рейтингов

1. Вычисляется количество отзывов для каждой оценки (1-5)
2. Рассчитывается процент для каждой оценки
3. Отображается визуально в виде прогресс-баров
4. Помогает пользователям быстро понять общее мнение

## Форматы данных

### Структура отзыва (от API)

```javascript
{
  id: 1,
  user_name: "Иван",
  rating: 5,
  comment: "Отличный товар! Очень доволен покупкой.",
  is_verified_purchase: true,
  is_edited: false,
  created_at: "2024-01-15T10:30:00Z",
  updated_at: "2024-01-15T10:30:00Z"
}
```

### Структура ответа API для списка отзывов

```javascript
{
  reviews: [...],
  rating: 4.5,
  reviews_count: 25,
  rating_distribution: {
    "5": 10,
    "4": 8,
    "3": 5,
    "2": 1,
    "1": 1
  },
  pagination: {
    page: 1,
    per_page: 10,
    total: 25,
    total_pages: 3
  },
  user_review: {
    id: 1,
    rating: 5,
    comment: "...",
    created_at: "..."
  }
}
```

### Структура ответа API для проверки возможности отзыва

```javascript
{
  can_review: true,
  has_review: false,
  purchase_date: "2024-01-10T14:20:00Z",
  message: "Вы можете оставить отзыв"
}
```

## Обработка ошибок

### 1. Ошибки загрузки

- Показывается сообщение об ошибке
- Возможность повторить запрос
- Graceful degradation (компоненты не ломают страницу)

### 2. Ошибки валидации

- Подсветка полей с ошибками
- Четкие сообщения об ошибках
- Предотвращение отправки невалидных данных

### 3. Ошибки прав доступа

- Сообщение о необходимости покупки
- Ссылка на страницу покупки (если применимо)
- Информативные сообщения
