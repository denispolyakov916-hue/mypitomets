Если вы обнаружили уязвимость безопасности в проекте Питомец+, пожалуйста, **НЕ** создавайте публичный issue. Вместо этого:

1. **Отправьте email** на адрес: [ВАШ EMAIL ДЛЯ БЕЗОПАСНОСТИ]
2. **Укажите в теме:** `[SECURITY] Уязвимость в Питомец+`
3. **Опишите проблему** детально с шагами воспроизведения

Мы ответим в течение 48 часов и будем держать вас в курсе прогресса исправления.

## 🛡️ Известные меры безопасности

### Backend (Django)
- Использование Django ORM для защиты от SQL-инъекций
- JWT токены для аутентификации
- Валидация всех входных данных
- CORS настроен только для доверенных доменов
- Argon2id для хэширования паролей

### Frontend (React)
- Валидация форм на клиенте
- Безопасное хранение токенов
- Защита от XSS через React
- Валидация API ответов

### Инфраструктура
- HTTPS в продакшене
- Ограничение количества запросов (rate limiting)
- Мониторинг и логирование

## 📋 Что считается уязвимостью

- SQL инъекции
- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)
- Аутентификация и авторизация
- Обход контроля доступа
- Утечка чувствительных данных
- Denial of Service (DoS)

## 🎯 Сроки реагирования

- **Критические уязвимости:** исправление в течение 7 дней
- **Высокий приоритет:** исправление в течение 14 дней
- **Средний приоритет:** исправление в следующем релизе
- **Низкий приоритет:** исправление при возможности
