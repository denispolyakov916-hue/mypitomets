# Ошибка белого экрана при переходе в корзину

## Описание проблемы

При переходе на страницу корзины (`/cart`) пользователь видел белый экран вместо содержимого корзины.

## Дата обнаружения

24 декабря 2025

## Причина ошибки

Ошибка была вызвана неправильным порядком определения функции в компоненте `Cart.jsx`.

### Техническая детализация

В файле `/frontend/src/pages/Shop/Cart.jsx` функция `isCourse` использовалась на строках 75-76:

```javascript
const products = items.filter(item => !isCourse(item))
const courses = items.filter(item => isCourse(item))
```

Но определение этой функции находилось гораздо ниже в коде (на строках 211-213):

```javascript
const isCourse = (item) => {
  return item.course !== undefined && item.course !== null
}
```

### Почему это вызывало ошибку?

В JavaScript функции, объявленные через `const` или `let`, **не поднимаются** (hoisting). Это означает, что они не могут быть использованы до их объявления в коде.

При попытке выполнить код:
```javascript
const products = items.filter(item => !isCourse(item))
```

JavaScript выбрасывал ошибку:
```
ReferenceError: Cannot access 'isCourse' before initialization
```

Эта ошибка приводила к падению компонента React, что и вызывало белый экран.

## Решение

### Изменения в коде

1. **Перемещено определение функции `isCourse` выше её первого использования**

   Функция была перемещена сразу после объявления хуков состояния, перед использованием в фильтрации:

   ```javascript
   const [redirectMessage, setRedirectMessage] = useState(null)
   const [isDeletingSelected, setIsDeletingSelected] = useState(false)

   /**
    * Проверка, является ли элемент курсом
    */
   const isCourse = (item) => {
     return item.course !== undefined && item.course !== null
   }

   // Разделение товаров и курсов
   const products = items.filter(item => !isCourse(item))
   const courses = items.filter(item => isCourse(item))
   ```

2. **Удалено дублирующееся определение функции**

   Старое определение функции `isCourse` (которое находилось ниже в коде) было удалено, чтобы избежать дублирования.

### Файлы, которые были изменены

- `/frontend/src/pages/Shop/Cart.jsx`

## Проверка согласованности API

В процессе исправления также была проверена согласованность формата данных между бэкендом и фронтендом:

### Бэкенд (Django REST Framework)

Эндпоинт: `GET /api/shop/cart/`

Возвращает:
```json
{
  "products": [...],
  "courses": [...],
  "totals": {
    "products": 0,
    "courses": 0,
    "total": 0
  },
  "items_count": 0,
  "can_checkout": false
}
```

### Фронтенд (React + Zustand)

В `cartStore.loadCart()` данные обрабатываются корректно:
```javascript
const products = response.products || []
const courses = response.courses || []
const allItems = [...products, ...courses]
```

**Вывод:** Формат данных согласован, проблема была только в порядке определения функции.

## Уроки на будущее

1. **Порядок объявления функций важен**
   - Функции, объявленные через `const`/`let`, должны быть определены до их использования
   - Альтернатива: использовать function declaration (поднимается), но это менее предпочтительно в React

2. **Проверка ошибок в консоли браузера**
   - Ошибки JavaScript всегда отображаются в консоли разработчика
   - При белом экране первым делом нужно проверить консоль браузера

3. **Использование ESLint**
   - Правила линтера могут помочь обнаружить такие проблемы на этапе разработки
   - Рекомендуется настроить правила для проверки порядка объявлений
