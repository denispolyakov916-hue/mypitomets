## Общая информация

### Цель рефакторинга
Полная стабилизация платформы, устранение всех известных багов, обеспечение корректного отображения данных во всех блоках системы, улучшение производительности и пользовательского опыта без удаления функциональности.

### Принципы рефакторинга
- **Не ломать работающее**: Все изменения должны быть обратно совместимы
- **Постепенность**: Поэтапное внедрение с тестированием на каждом этапе
- **Документирование**: Все изменения должны быть задокументированы
- **Безопасность**: Приоритет безопасности данных и пользователей
- **Производительность**: Оптимизация без потери функциональности

### Текущее состояние системы

#### Backend
- **Django**: 4.2.8 (требует обновления до 5.1.x)
- **Django REST Framework**: 3.14.0
- **PostgreSQL**: 15+
- **JWT**: djangorestframework-simplejwt 5.3.0
- **Python**: 3.10+

#### Frontend
- **React**: 18.2.0
- **Vite**: 5.0.0
- **React Router**: 6.20.0
- **Zustand**: 4.4.7
- **Node.js**: 18+

**1.3.3 Добавить валидацию в сериализаторы**
- Обновить все сериализаторы с JSON полями
- Добавить кастомные `validate_*` методы
- Добавить проверки типов данных

**Пример валидатора**:
```python
# backend/core/validators.py
from django.core.exceptions import ValidationError

def validate_json_list(value, allowed_values=None):
    """Валидация JSON списка."""
    if not isinstance(value, list):
        raise ValidationError("Значение должно быть списком")
    if allowed_values:
        for item in value:
            if item not in allowed_values:
                raise ValidationError(f"Недопустимое значение: {item}")
    return value
```

**Результат**:
