# Часть I. Описание

## 1. Назначение документа

Настоящий документ определяет требования к составу, формату и качеству данных по кормам, лакомствам и добавкам для кошек и собак, которые необходимы для работы сервиса персонализированного подбора питания.

Документ предназначен для руководства и продуктовой команды партнёра-поставщика данных. Цель — согласовать единый формат передачи данных и зафиксировать, **какие именно данные с этикетки продукта** должны быть переданы в структурированном виде.

## 2. Проблематика

### 2.1. Незнание владельца

Подбор питания для питомца — задача, требующая специальных знаний. Абсолютное большинство владельцев не обладают компетенциями в области ветеринарной диетологии. На практике выбор корма происходит "на глаз": по цене, по акции, по совету знакомых, по привлекательности упаковки. Критерии, действительно важные для здоровья питомца — калорийность, баланс нутриентов, состав, наличие аллергенов — при таком подходе не учитываются вообще.

### 2.2. Отложенные последствия

Ключевая проблема в том, что ошибка в питании проявляется **не сразу**. Владелец может месяцами кормить питомца неподходящим рационом и не замечать этого. Первые симптомы (проблемы с ЖКТ, шерстью, кожей, весом, мочевыделительной системой) появляются постепенно. Как правило, владелец узнаёт о том, что кормил питомца неправильно, только **на приёме у ветеринара**, когда здоровье уже пострадало.

Типичная ситуация: владелец длительное время кормит кота неподходящим рационом. Проблемы со здоровьем накапливаются. Ветеринар корректирует питание — и состояние питомца нормализуется. Владелец впоследствии осознаёт, что проблему можно было предотвратить, если бы он изначально выбрал правильный корм.

### 2.3. Недоверие к рекомендациям

Когда рекомендации по кормам не подкреплены объективными данными, пользователи воспринимают их как маркетинг. Это снижает доверие к любым сервисам подбора питания. Чтобы рекомендация воспринималась как надёжная, она должна быть **объяснимой**: пользователь должен видеть, почему именно этот корм ему рекомендован, на основании каких характеристик (состав, калорийность, нутриенты, нормы кормления).

## 3. Концепция сервиса

Наш сервис решает описанную проблему: автоматизирует "ветеринарный здравый смысл" и делает его доступным массовому пользователю.

**Как это работает (в общих чертах):**

1. **Профиль питомца.** Пользователь вводит данные о питомце: вид (кошка/собака), порода, возраст, масса тела, уровень активности, состояние здоровья (стерилизация, аллергии, хронические заболевания, особенности).

2. **Расчёт потребностей.** На основании профиля сервис рассчитывает индивидуальную суточную потребность питомца: энергия (ккал), белки, жиры, клетчатка, ключевые микроэлементы.

3. **Сопоставление с кормами.** Рассчитанные потребности сопоставляются с характеристиками кормов из базы данных: нутриентный состав, ингредиенты, наличие аллергенов, специальные назначения, нормы кормления.

4. **Персональная подборка.** Пользователь получает подборку кормов, ранжированную по степени соответствия потребностям его питомца, с рекомендованной суточной порцией в граммах.

## 4. Роль данных по кормам

Качество персонализации **напрямую зависит** от полноты и точности данных о продуктах.

### 4.1. Почему нужны именно данные с этикетки

- **Разные питомцы — разные потребности.** Возраст, масса, активность и состояние здоровья радикально меняют оптимальный рацион. Без точных данных о составе корма невозможно определить, подходит ли он конкретному питомцу.

- **Схожие названия ≠ схожий состав.** Два корма "с курицей" от разных производителей могут иметь совершенно разное соотношение белков, жиров, калорийности, процентов мясного компонента и добавок. Маркетинговое название не несёт информации, достаточной для подбора.

- **Нормы кормления — основа для расчёта порции.** Перевод суточной потребности (ккал/сутки) в конкретные граммы возможен только при наличии данных о калорийности и рекомендованных нормах кормления для каждого продукта.

- **Ингредиенты и аллергены — основа для исключений.** Если у питомца аллергия на курицу или непереносимость зерновых, сервис должен исключить все корма, содержащие эти компоненты. Для этого нужен полный структурированный состав.

- **Доверие пользователя.** Когда рекомендация объяснима ("этот корм подходит, потому что содержит 30% белка, не содержит зерновых, калорийность 380 ккал/100г — для вашего кота это оптимально"), пользователь воспринимает сервис как надёжный инструмент, а не как рекламу.

### 4.2. Что именно нужно от партнёра

Нам нужен **регулярный экспорт структурированных данных** по всем SKU кормов, лакомств и добавок для кошек и собак.

Принцип простой: **всё, что указано на этикетке продукта и влияет на питание, должно быть передано в структурированном виде.**

Конкретно:
- Идентификация продукта (бренд, название, тип, вид животного, возрастная группа).
- Нутриентный состав (БЖУ, калорийность, зола, влажность, клетчатка) — на 100 г.
- Полный список ингредиентов с порядком и процентами (если указаны).
- Витамины, минералы, добавки — с указанием значений и единиц измерения.
- Нормы кормления (таблица по весу животного → грамм/сутки).
- Признаки специального назначения (стерилизованные, гипоаллергенный, контроль веса и т.д.).
- Параметры упаковки (вес, тип).

# Часть II. Требования к данным

## 5. Формат поставки

### 5.1. Формат файла

| Параметр | Значение |
|---|---|
| Формат | JSON Lines (`.jsonl`): 1 строка = 1 SKU. Допускается JSON Array (`.json`) |
| Кодировка | UTF-8 |
| Пустые значения | `null` для скаляров, `[]` для массивов |

### 5.2. Единицы измерения

| Величина | Единица | Примечание |
|---|---|---|
| Нутриенты | на 100 г продукта | Если на упаковке иначе — передать как есть + пояснение в `notes` |
| Вес упаковки | граммы (`g`) | Если на упаковке кг — переводить: 1.5 кг → 1500 |
| Калорийность | ккал/100 г | Поле `energy_kcal_per_100g` |
| Проценты | число 0–100 | Например: белок 30% → `30` |
| Витамины/минералы | как на упаковке | IU, mg, mcg — передавать единицу в поле `unit` |

### 5.3. Приоритет источника данных

Если значение указано **на упаковке** продукта, оно имеет приоритет над данными из внутренних систем, с сайта или из маркетинговых описаний.

### 5.4. Обновления

- Каждая запись содержит `updated_at` — дата/время последнего обновления данных.
- Идентификаторы `product_id` и `sku_id` **не должны меняться** при обновлениях. Это идентификаторы партнёра, мы используем их для сопоставления.
- При изменении состава, нутриентов или норм кормления — обновить соответствующие поля и `updated_at`.

Если на упаковке указан возраст текстом или диапазоном:

### Кошки

| Значение | Возраст | Ключевые слова на упаковке |
|---|---|---|

| Значение | Возраст | Ключевые слова на упаковке |
|---|---|---|

| `adult` | 12(18)–84 мес | Adult, Adult 1+, 1-7 лет |
| `senior` | 84+ мес | Senior, Mature 7+, 8+ |
| `all` | — | "Для всех возрастов", "All life stages" |

Если возраст не указан: `life_stage = "all"`, `age_range_months = null`.

## 9. Правила заполнения

| Правило | Описание |
|---|---|
| Нутриенты | Всегда на 100 г. Если на упаковке иначе — передать как есть + пояснение |
| Универсальный товар | `life_stage = "all"`, `breed = null`, `breed_size = "all"` |
| Оси вариаций SKU | Только **вес** и **вкус** |
| Отсутствующие данные | `null` для скаляров, `[]` для массивов |
| Приоритет | Данные с упаковки > данные из внутренних систем |

## 10. Контроль качества

### 10.1. Обязательные проверки перед передачей

- [ ] Нет пустых / битых JSON-объектов.
- [ ] У каждой записи заполнены: `product_id`, `sku_id`, `animal_type`, `product_type`, `product_title`, `brand`, `updated_at`.
- [ ] Для `product_type = "food"`: заполнены `nutrition` (минимум: калорийность, белок, жир) и `ingredients`.
- [ ] Для `product_type = "supplement"`: заполнены `ingredients` и/или `vitamins` / `minerals` / `additives`.
- [ ] Для `product_type = "treat"`: заполнены `ingredients`.
- [ ] Для всех типов с нормами кормления на упаковке: заполнен `feeding_guidelines[]` или `feeding_guidelines_text`.

### 10.2. Минимальные требования по подкатегориям

| Тип | Обязательные блоки |
|---|---|
| `food` (корм) | `nutrition` + `ingredients` + `feeding_guidelines` (или `_text`) |
| `treat` (лакомство) | `ingredients` + `nutrition` (если на упаковке есть) |
| `supplement` (добавки) | `ingredients` и/или `vitamins` / `minerals` / `additives` |

```

Название:               Сухой корм для взрослых кошек с курицей
Тип:                    Полнорационный корм
Вид животного:          Кошки
Возраст:                Взрослые (1-7 лет)
Масса нетто:            400 г
Упаковка:               Пакет

Состав: курица (30%), рис, животный жир, кукурузный глютен,
свекольный жом, рыбий жир, дрожжи, минеральные вещества.

Аналитические компоненты (на 100 г):
  Белок ............. 30 г
  Жир ............... 12 г
  Клетчатка ......... 3 г
  Зола .............. 7 г
  Влажность ......... 10%

Энергетическая ценность: 380 ккал / 100 г

Добавки (на кг):
  Витамин A ......... 15 000 IU
  Цинк ............. 120 mg
  Таурин ........... 1 000 mg

Нормы кормления:
  Вес кошки 3 кг — 45 г/сутки
  Вес кошки 4 кг — 60 г/сутки
  Вес кошки 5 кг — 70 г/сутки

Назначение: для стерилизованных кошек
Страна: Россия
```
