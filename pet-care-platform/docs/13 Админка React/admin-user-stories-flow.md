## 2. User Stories

### 2.1 Функция: Аутентификация администратора

```
Функция: Аутентификация администратора

Как администратор (staff user),
Я хочу войти в админ-панель с помощью email и пароля,
Чтобы получить доступ к управлению платформой.

Критерии приёмки:
- На странице /admin-panel/login отображается форма входа с полями email и пароль
- При успешной аутентификации происходит redirect на /admin-panel/dashboard
- При неудачной аутентификации показывается понятная ошибка на русском языке
- Токен сохраняется и используется для последующих запросов
- При отсутствии прав staff показывается сообщение "У вас нет прав администратора"
- Форма входа имеет визуальное отличие от обычной формы входа (брендинг админки)

Сценарий: Успешный вход администратора
  Дано Я нахожусь на странице /admin-panel/login
  Когда Я ввожу корректный email и пароль пользователя с is_staff=True
  И нажимаю кнопку "Войти в админ-панель"
  Тогда Меня перенаправляет на /admin-panel/dashboard
  И В header отображается мой email и роль
  И Sidebar показывает доступные мне разделы

Сценарий: Неудачный вход (неправильные credentials)
  Дано Я нахожусь на странице /admin-panel/login
  Когда Я ввожу некорректный email или пароль
  И нажимаю кнопку "Войти в админ-панель"
  Тогда Я остаюсь на странице входа
  И Вижу сообщение об ошибке "Неверный email или пароль"
  И Поле пароля очищается
  И Фокус остаётся на форме

Сценарий: Попытка входа без прав staff
  Дано Я нахожусь на странице /admin-panel/login
  Когда Я ввожу email и пароль пользователя с is_staff=False
  И нажимаю кнопку "Войти в админ-панель"
  Тогда Я остаюсь на странице входа
  И Вижу сообщение "У вас нет прав для доступа к админ-панели"
  И Вижу ссылку "Вернуться на главную"

Сценарий: Переход на защищённую страницу без авторизации
  Дано Я не авторизован
  Когда Я перехожу на /admin-panel/dashboard
  Тогда Я вижу страницу входа /admin-panel/login
  И Вижу сообщение "Для доступа необходимо авторизоваться"
  И После успешного входа меня перенаправит на /admin-panel/dashboard
```

### 2.2 Функция: Выход из админ-панели

```
Функция: Выход из админ-панели

Как администратор,
Я хочу безопасно выйти из админ-панели,
Чтобы защитить данные при работе на общем компьютере.

Критерии приёмки:
- Кнопка выхода находится в header и легко доступна
- При выходе очищаются все токены и данные сессии
- После выхода происходит redirect на страницу входа админки
- Невозможно вернуться в админку без повторной авторизации (кнопка назад)

Сценарий: Успешный выход
  Дано Я авторизован как администратор
  И Я нахожусь на любой странице админ-панели
  Когда Я нажимаю кнопку "Выйти" в header
  Тогда Я вижу страницу входа /admin-panel/login
  И Вижу сообщение "Вы успешно вышли из системы"
  И Токены удалены из localStorage

Сценарий: Попытка вернуться после выхода
  Дано Я вышел из админ-панели
  Когда Я нажимаю кнопку "Назад" в браузере
  Тогда Я вижу страницу входа /admin-panel/login
  И Не получаю доступ к защищённым данным
```

### 2.3 Функция: Просмотр дашборда

```
Функция: Просмотр дашборда

Как администратор,
Я хочу видеть ключевые метрики платформы на главной странице,
Чтобы быстро оценить состояние бизнеса.

Критерии приёмки:
- Дашборд показывает метрики: пользователи, заказы, выручка, питомцы
- Данные загружаются автоматически при входе
- При ошибке загрузки показывается понятное сообщение
- Есть возможность обновить данные вручную
- Дашборд адаптивен для разных размеров экрана

Сценарий: Успешная загрузка дашборда
  Дано Я авторизован как администратор
  Когда Я перехожу на /admin-panel/dashboard
  Тогда Я вижу карточки с метриками:
    | Метрика | Описание |
    | Пользователи | Общее количество и новые за период |
    | Заказы | Количество и сумма за период |
    | Товары | Активные товары и бестселлеры |
    | Курсы | Активные курсы и записи |
  И Вижу графики трендов
  И Данные обновляются каждые 5 минут

Сценарий: Ошибка загрузки данных
  Дано Я авторизован как администратор
  И Сервер недоступен
  Когда Я перехожу на /admin-panel/dashboard
  Тогда Я вижу сообщение "Не удалось загрузить данные"
  И Вижу кнопку "Повторить попытку"
```

### 2.4 Функция: Управление заказами

```
Функция: Управление заказами

Как менеджер заказов,
Я хочу просматривать и обрабатывать заказы клиентов,
Чтобы обеспечить своевременную доставку.

Критерии приёмки:
- Таблица заказов с фильтрацией по статусу, дате, пользователю
- Возможность изменить статус заказа
- Просмотр деталей заказа в модальном окне
- Массовое обновление статусов
- Экспорт заказов в CSV/Excel

Сценарий: Просмотр списка заказов
  Дано Я авторизован как менеджер заказов
  Когда Я перехожу на /admin-panel/orders
  Тогда Я вижу таблицу заказов с колонками:
    | ID | Клиент | Сумма | Статус | Дата | Действия |
  И Вижу фильтры по статусу и дате
  И Вижу пагинацию

Сценарий: Изменение статуса заказа
  Дано Я нахожусь на странице заказов
  И Вижу заказ со статусом "Ожидает оплаты"
  Когда Я нажимаю на заказ
  И Выбираю новый статус "Оплачен"
  И Нажимаю "Сохранить"
  Тогда Статус заказа обновляется
  И Я вижу уведомление "Статус заказа обновлён"

Сценарий: Массовое обновление статусов
  Дано Я нахожусь на странице заказов
  Когда Я выбираю несколько заказов чекбоксами
  И Выбираю действие "Изменить статус на Отправлен"
  И Подтверждаю действие
  Тогда Все выбранные заказы получают новый статус
  И Я вижу уведомление "Обновлено N заказов"
```

### 2.5 Функция: Управление курсами

```
Функция: Управление курсами

Как контент-менеджер,
Я хочу создавать и редактировать курсы обучения,
Чтобы предоставлять качественный контент пользователям.

Критерии приёмки:
- Таблица курсов с поиском и фильтрацией
- Переход в конструктор курсов
- Возможность активировать/деактивировать курс
- Просмотр статистики записей на курс

Сценарий: Просмотр списка курсов
  Дано Я авторизован как контент-менеджер
  Когда Я перехожу на /admin-panel/courses
  Тогда Я вижу таблицу курсов с колонками:
    | Название | Категория | Цена | Статус | Записей | Действия |
  И Вижу кнопку "Создать курс"

Сценарий: Переход в конструктор курса
  Дано Я нахожусь на странице курсов
  Когда Я нажимаю на курс или кнопку "Редактировать"
  Тогда Я перехожу на /admin/courses/:id/builder
  И Вижу конструктор курсов с drag-and-drop

Сценарий: Активация/деактивация курса
  Дано Я нахожусь на странице курсов
  Когда Я переключаю статус курса
  Тогда Курс становится активным/неактивным
  И Я вижу уведомление об изменении
```

### 2.6 Функция: Экспорт данных

```
Функция: Экспорт данных

Как администратор,
Я хочу экспортировать данные в различных форматах,
Чтобы использовать их для отчётности и анализа.

Критерии приёмки:
- Поддержка форматов: CSV, Excel, PDF, JSON
- Возможность выбора периода и фильтров
- Показ прогресса экспорта для больших данных
- Автоматическое скачивание файла

Сценарий: Экспорт заказов в Excel
  Дано Я нахожусь на странице заказов
  Когда Я нажимаю кнопку "Экспорт"
  И Выбираю формат "Excel"
  И Выбираю период "Последние 30 дней"
  И Нажимаю "Скачать"
  Тогда Начинается скачивание файла orders_2025-12-27.xlsx
  И Файл содержит заказы за выбранный период
```

## 4. Состояния интерфейса

### 4.1 Страница входа (/admin-panel/login)

| Состояние | Описание | UI элементы |
|-----------|----------|-------------|
| **Initial** | Пустая форма | Email, Password, Button "Войти" |
| **Loading** | Отправка данных | Button disabled, Spinner |
| **Error: credentials** | Неверные данные | Error message, Form reset |
| **Error: no_staff** | Нет прав | Error message, Link to home |
| **Error: network** | Нет сети | Error message, Retry button |
| **Success** | Успешный вход | Redirect to dashboard |

### 4.2 Дашборд (/admin-panel/dashboard)

| Состояние | Описание | UI элементы |
|-----------|----------|-------------|
| **Loading** | Загрузка данных | Skeleton cards, Skeleton charts |
| **Success** | Данные загружены | Metric cards, Charts, Tables |
| **Error** | Ошибка загрузки | Error message, Retry button |
| **Partial** | Часть данных | Loaded cards, Error for failed |
| **Refreshing** | Обновление | Data + subtle loader |

### 4.3 Таблицы данных

| Состояние | Описание | UI элементы |
|-----------|----------|-------------|
| **Loading** | Первая загрузка | Table skeleton |
| **Empty** | Нет данных | Empty state illustration |
| **Data** | Есть данные | Table rows, Pagination |
| **Filtering** | Применение фильтра | Dimmed table, Loader |
| **Error** | Ошибка | Error message in table |

## 6. Приоритеты реализации

### 6.1 MVP (Минимально жизнеспособный продукт)

| Приоритет | Функция | Описание |
