# Техническое задание: База данных по породам

## 1. Общие сведения

### 1.1. Наименование проекта
База знаний характеристик пород собак и кошек для платформы "Питомец+"

### 1.2. Цель проекта
Создать комплексную систему хранения и анализа характеристик пород для персонализации всех услуг платформы и улучшения качества рекомендаций пользователям.

### 1.3. Задачи проекта
- Разработать структуру данных для хранения характеристик пород
- Реализовать API для доступа к данным пород
- Создать систему сравнения параметров питомца с эталоном породы
- Интегрировать базу пород со всеми модулями платформы (PetID, Магазин, Курсы, Календарь)
- Обеспечить автоматическое обновление и валидацию данных

## 2. Требования к системе

### 2.1. Функциональные требования

#### 2.1.1. Хранение данных пород
- **FR-1.1**: Система должна хранить характеристики  пород (58 кошек, 66 собак)
- **FR-1.2**: Данные должны включать физические параметры, поведение, здоровье, уход
- **FR-1.3**: Система должна поддерживать связь пород с генетическими рисками здоровья
- **FR-1.4**: Должны храниться рекомендации по питанию для каждой породы
- **FR-1.5**: Система должна содержать специфические процедуры ухода по породам

#### 2.1.2. API для работы с породами
- **FR-2.1**: GET /api/pets/breeds/ - получение списка пород с фильтрацией
- **FR-2.2**: GET /api/pets/breeds/{slug}/ - получение детальной информации о породе
- **FR-2.3**: GET /api/pets/{pet_id}/breed-comparison/ - сравнение питомца с эталоном породы
- **FR-2.4**: GET /api/pets/breeds/{id}/health/ - получение рисков здоровья породы
- **FR-2.5**: Поддержка фильтров: species, size_category, hypoallergenic, apartment_friendly

#### 2.1.3. Сравнение с питомцем
- **FR-3.1**: Анализ веса питомца относительно стандарта породы (±15% допуск)
- **FR-3.2**: Оценка уровня активности (низкий/средний/высокий)
- **FR-3.3**: Анализ совместимости поведения
- **FR-3.4**: Оценка рисков здоровья с учетом возраста
- **FR-3.5**: Проверка условий содержания (квартира/дом/двор)
- **FR-3.6**: Генерация персонализированных рекомендаций (балл 0-100)

#### 2.1.4. Интеграция с другими модулями
- **FR-4.1**: PetID - автозаполнение профиля, сравнение параметров
- **FR-4.2**: Магазин - подбор корма и товаров по характеристикам породы
- **FR-4.3**: Курсы - рекомендации по дрессировке на основе обучаемости
- **FR-4.4**: Календарь - генерация напоминаний по уходу
- **FR-4.5**: Аналитика - статистика по популярности пород

### 2.2. Нефункциональные требования

#### 2.2.1. Производительность
- **NFR-1.1**: Время ответа API < 200ms для основных запросов
- **NFR-1.2**: Поддержка одновременных запросов 1000+ пользователей
- **NFR-1.3**: Кэширование часто запрашиваемых данных

#### 2.2.2. Надежность
- **NFR-2.1**: Доступность системы 99.9%
- **NFR-2.2**: Валидация входных данных
- **NFR-2.3**: Graceful degradation при недоступности части данных

#### 2.2.3. Масштабируемость
- **NFR-3.1**: Легкое добавление новых пород и характеристик
- **NFR-3.2**: Поддержка расширения на другие виды животных
- **NFR-3.3**: Оптимизированные запросы к БД с индексами

## 3. Архитектура системы

### 3.1. Структура данных

```
Breed (Основная модель)
├── id: Integer (PK, FK из JSON)
├── Основные характеристики (25 полей)
├── health_risks: ForeignKey → BreedHealth[]
├── nutrition: OneToOne → BreedNutrition
└── care_procedures: ForeignKey → BreedCare[]

BreedHealth (Риски здоровья)
├── condition_name: Название заболевания
├── severity: Тяжесть (low/medium/high)
├── prevalence_percent: Распространенность
├── affected_system: Затронутая система
└── prevention/screening: Профилактика/обследования

BreedNutrition (Рекомендации по питанию)
├── protein_need: Потребность в белке
├── calorie_density: Калорийность
├── diet_type: Тип питания
└── special_considerations: Особые рекомендации

BreedCare (Процедуры ухода)
├── care_category: Категория ухода
├── procedure: Процедура
├── frequency: Частота
└── importance: Важность
```

### 3.2. API архитектура

#### Основные эндпоинты
```
GET  /api/pets/breeds/           # Список с фильтрами
GET  /api/pets/breeds/{slug}/    # Детали породы
GET  /api/pets/{pet_id}/breed-comparison/  # Сравнение
POST /api/pets/breeds/_search    # Поиск пород
```

#### Структура ответа сравнения
```json
{
  "pet": { /* данные питомца */ },
  "breed_standard": { /* эталон породы */ },
  "analysis": {
    "weight": { "status": "normal", "score": 95 },
    "activity": { "status": "normal", "score": 88 },
    "health": { "status": "attention", "score": 75 },
    "housing": { "status": "good", "score": 100 }
  },
  "health_risks": [ /* топ-5 рисков */ ],
  "recommendations": [ /* персонализированные советы */ ],
  "overall_score": 89
}
```

### 3.3. Сервисный слой

#### PetBreedComparisonService
- `compare_pet_with_breed(pet)` - комплексное сравнение
- `analyze_weight(pet, breed)` - анализ веса
- `analyze_health_risks(pet, breed)` - оценка рисков
- `generate_recommendations(pet, breed)` - генерация советов

#### Ключевые алгоритмы
- **Сравнение веса**: ±15% от эталона = нормальный вес
- **Оценка активности**: маппинг уровней энергии
- **Расчет общего скора**: взвешенная сумма (вес 20%, активность 15%, здоровье 20% и т.д.)
