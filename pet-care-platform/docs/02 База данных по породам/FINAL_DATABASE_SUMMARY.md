## 📋 КРАТКОЕ ОПИСАНИЕ КАЖДОЙ ТАБЛИЦЫ

### 1. **breeds.json** (128 пород)
**Что:** Основные характеристики пород  
**Зачем:** Эталон для сравнения с PetID, подбор корма, курсов  
**Поля:** 25 (размеры, поведение, уход, здоровье)

### 2. **breed_health.json** (649 записей)
**Что:** Генетические риски здоровья  
**Зачем:** Предупреждения, скрининги, подбор кормов для профилактики  
**Поля:** 9 (название, тип, система, тяжесть, профилактика)

### 3. **breed_nutrition.json** (55 записей)
**Что:** Рекомендации по питанию  
**Зачем:** Расчет рациона, подбор корма  
**Поля:** 6 (белок, калории, тип корма, частота)

### 4. **breed_care.json** (32 записи)
**Что:** Специфичные процедуры ухода  
**Зачем:** Персонализированные напоминания  
**Поля:** 7 (категория, процедура, частота, важность)

### 5. **common_data.json** (общие данные)
**Что:** Данные для ВСЕХ пород  
**Зачем:** Исключение дублирования, стандартные напоминания  
**Разделы:** Вакцинации, уход, паразиты

## 🎯 ИСПОЛЬЗОВАНИЕ В СЕРВИСАХ

### 1. Магазин (персонализация товаров)
**Используемые поля:**
- `breeds.size_category` → размер порций, аксессуаров
- `breeds.energy_level` → калорийность корма
- `breeds.brachycephalic` → специальные миски
- `breed_health.condition_name` → корма для профилактики
- `breed_nutrition.protein_need` → содержание белка в корме

### 2. Подбор корма (расчет рациона)
**Используемые поля:**
- `breeds.weight_min/max` → идеальный вес
- `breeds.energy_level` → базовая активность
- `breed_nutrition.*` → все поля
- `breed_health.*` → специализированные корма

### 3. PetID (сравнение с эталоном)
**Сравниваемые параметры:**
- Вес (pet.weight vs breeds.weight_min/max)
- Активность (pet.activity_level vs breeds.energy_level)
- Здоровье (pet.health_issues vs breed_health.*)

### 4. Дневник здоровья (напоминания)
**Генерация событий:**
- Вакцинации (из common_data.vaccinations)
- Уход (из breed_care.* + common_data.care_procedures)
- Скрининги (из breed_health.screening)
- Паразиты (из common_data.parasite_prevention)
