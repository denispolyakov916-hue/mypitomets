## Критические баги

### 🔴 BUG-001: Белый экран при переходе в корзину
**Статус**: ✅ Исправлено (24 декабря 2025)  
**Приоритет**: Критический  
**Файл**: `frontend/src/pages/Shop/Cart.jsx`

**Описание**:  
При переходе на страницу корзины (`/cart`) пользователь видел белый экран вместо содержимого корзины.

**Причина**:  
Неправильный порядок определения функции `isCourse` - функция использовалась до её объявления.

**Решение**:  
Перемещено определение функции `isCourse` выше её первого использования.

**Ссылка**: `docs/troubleshooting/белый_экран_ошибка.md`

---

### 🔴 BUG-002: Отсутствие загрузки профиля при старте
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Критический  
**Файл**: `frontend/src/main.jsx`, `frontend/src/store/authStore.js`

**Описание**:  
При refresh страницы `user=null`, права не проверяются. Пользователь может потерять доступ к защищенным страницам.

**Влияние**:  
- Потеря контекста авторизации
- Проблемы с доступом к защищенным маршрутам
- Проблемы в админ-панели

**Решение**:  
✅ Улучшена инициализация в `main.jsx` - теперь проверяет токен в localStorage при старте и загружает профиль через `validateToken()`.

**Ссылка**: `docs/admin/admin-panel-context.md` (строка 110)

---

### 🔴 BUG-003: Неправильный redirect после входа
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Критический  
**Файл**: `frontend/src/pages/Auth/Login.jsx`

**Описание**:  
После входа редирект на `/pets` вместо `/admin-panel` для администраторов.

**Влияние**:  
Администраторы не попадают в админ-панель после входа.

**Решение**:  
✅ Добавлена проверка `is_staff` и `is_superuser` в `Login.jsx`. Администраторы теперь редиректятся на `/admin/dashboard`, обычные пользователи на `/pet-id`.

**Ссылка**: `docs/admin/admin-panel-context.md` (строка 108)

## Высокоприоритетные баги

### 🟠 BUG-004: Нет страницы входа для админов
**Статус**: ✅ Исправлено (существует AdminLoginPage)  
**Приоритет**: Высокий  
**Файл**: `frontend/src/admin/components/Auth/AdminLoginPage.jsx`

**Описание**:  
Пользователь не понимает куда попал и что нужны права staff для доступа к админ-панели.

**Влияние**:  
Плохой UX для администраторов.

**Решение**:  
✅ `AdminLoginPage` уже существует и используется в `AdminRoute`. Показывает понятные сообщения об ошибках и требованиях к доступу.

**Ссылка**: `docs/admin/admin-panel-context.md` (строка 107)

---

### 🟠 BUG-005: Двойная проверка авторизации
**Статус**: ⚠️ Требует проверки  
**Приоритет**: Высокий  
**Файл**: `frontend/src/components/AdminRoute.jsx`, `frontend/src/admin/App.jsx`

**Описание**:  
AdminRoute и AdminApp проверяют авторизацию отдельно, возможна race condition.

**Влияние**:  
Проблемы с синхронизацией состояния авторизации.

**Решение**:  
Унифицировать проверку авторизации в одном месте.

**Ссылка**: `docs/admin/admin-panel-context.md` (строка 109)

---

### 🟠 BUG-006: Отсутствие валидации JSON полей
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Высокий  
**Файлы**: `backend/core/validators.py`, все модели с JSON полями

**Описание**:  
JSON поля в моделях не имеют валидации структуры данных. Возможны некорректные данные в БД.

**Влияние**:  
- Некорректные данные в БД
- Ошибки при обработке данных
- Проблемы с отображением

**Решение**:  
✅ Созданы валидаторы в `backend/core/validators.py` и добавлены во все модели с JSON полями. Валидация работает на уровне модели.

**Ссылка**: `docs/JSON_VALIDATION.md`

---

### 🟠 BUG-007: N+1 проблемы в запросах
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Высокий  
**Файлы**: Все `views.py` файлы

**Описание**:  
Некоторые запросы имеют N+1 проблемы, что приводит к медленной работе API.

**Влияние**:  
- Медленная работа API
- Высокая нагрузка на БД
- Плохой UX

**Решение**:  
✅ Добавлены `select_related` и `prefetch_related` во все критические views. Оптимизированы запросы для продуктов, заказов, курсов, питомцев и админ API.

**Ссылка**: `docs/QUERY_OPTIMIZATION.md`

## Среднеприоритетные баги

**Статус**: ⚠️ Требует миграции  
**Приоритет**: Средний  
**Файлы**: `backend/apps/training/models.py`, `backend/apps/training/views.py`

**Описание**:  
Существуют две архитектуры курсов:

**Влияние**:

- Сложность поддержки
- Путаница в коде

**Решение**:  
Мигрировать все курсы на новую архитектуру и удалить старый код (или оставить для обратной совместимости).

### 🟡 BUG-009: Отсутствие обработки ошибок в некоторых компонентах
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Средний  
**Файлы**: `frontend/src/utils/errorHandler.js`, `frontend/src/utils/retry.js`, `frontend/src/components/ErrorBoundary.jsx`

**Описание**:  
Не все компоненты обрабатывают ошибки API запросов.

**Влияние**:  
- Белые экраны при ошибках
- Плохой UX
- Сложность отладки

**Решение**:  
✅ Созданы утилиты для обработки ошибок (`errorHandler.js`, `retry.js`). Улучшен `ErrorBoundary` компонент. Рекомендуется использовать эти утилиты во всех компонентах.

**Ссылка**: `docs/FRONTEND_ERROR_HANDLING.md`

---

### 🟡 BUG-010: Проблемы с типизацией
**Статус**: ✅ Частично исправлено (Январь 2026)  
**Приоритет**: Средний  
**Файлы**: Все React компоненты

**Описание**:  
Не все компоненты имеют PropTypes или TypeScript типы.

**Влияние**:  
- Ошибки типов во время выполнения
- Сложность поддержки
- Отсутствие автодополнения в IDE

**Решение**:  
✅ Добавлены PropTypes в ключевые компоненты (CourseCard, ProductCard, PetCard, Loader, Rating, FavoriteButton). Создан `utils/propTypes.js` с общими типами. Планируется миграция на TypeScript.

**Ссылка**: `docs/TYPING.md`

---

### 🟡 BUG-011: Неоптимальная загрузка компонентов
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Средний  
**Файл**: `frontend/src/App.jsx`, `frontend/vite.config.js`

**Описание**:  
Не все тяжелые компоненты загружаются лениво.

**Влияние**:  
- Медленная первая загрузка
- Большой размер бандла

**Решение**:  
✅ Реализован lazy loading для некритичных страниц (UnifiedCheckout, PaymentMethodSelection, Settings, HealthDiary, Orders, OrderDetail, PetIdPage). Настроен code splitting в Vite с разделением на vendor chunks.

**Ссылка**: `docs/BUNDLE_OPTIMIZATION.md`

## Технические проблемы

### TECH-001: Устаревшие версии зависимостей
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Высокий

**Проблема**:  
- Django 4.2.8 (актуальная версия 5.1.x)
- Возможно другие устаревшие зависимости

**Решение**:  
✅ Обновлен Django до 5.1.5 (LTS), DRF до 3.15.2, и другие зависимости до актуальных версий.

**Ссылка**: `docs/DJANGO_5_UPGRADE_GUIDE.md`, `docs/DEPENDENCIES_ANALYSIS.md`

---

### TECH-002: Отсутствие кэширования
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Средний

**Проблема**:  
Нет кэширования для тяжелых запросов.

**Решение**:  
✅ Настроено кэширование (LocMemCache для dev, Redis для prod). Добавлено кэширование для списков товаров, курсов, деталей и статистики. Реализована автоматическая инвалидация через signals.

**Ссылка**: `docs/CACHING.md`

---

### TECH-003: Недостаточное логирование
**Статус**: ✅ Исправлено (Январь 2026)  
**Приоритет**: Средний

**Проблема**:  
Не все критические операции логируются.

**Решение**:  
✅ Настроено структурированное логирование с файловыми и консольными обработчиками. Добавлен `RequestLoggingMiddleware` для логирования всех запросов. Реализован `SensitiveDataFilter` для фильтрации чувствительных данных.

**Ссылка**: `docs/LOGGING.md`
