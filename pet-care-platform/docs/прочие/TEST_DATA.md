## Обзор

Тестовые данные используются для:
- Разработки и тестирования функционала
- Демонстрации возможностей платформы
- Тестирования производительности
- Проверки корректности работы системы

Все тестовые данные должны быть валидными и соответствовать форматам, описанным в документации.

  - `support@pitometsplus.ru` - Специалист поддержки (пароль: support123)

  - Полные профили с именами, телефонами, городами, датами рождения

### generate_chart_test_data

**Расположение**: `backend/apps/analytics/management/commands/generate_chart_test_data.py`

**Назначение**: Генерация тестовых данных для аналитики и графиков

**Параметры**:
- `--days N` - Количество дней для генерации данных (по умолчанию: 90)
- `--users N` - Количество пользователей (по умолчанию: 50)
- `--pets N` - Количество питомцев (по умолчанию: 80)
- `--orders N` - Количество заказов (по умолчанию: 200)

**Использование**:
```bash
# Генерация данных за 90 дней
python manage.py generate_chart_test_data

# Генерация данных за 180 дней с большим количеством записей
python manage.py generate_chart_test_data --days=180 --users=100 --orders=500
```

**Создаёт**:
- Пользователей с распределёнными датами создания
- Питомцев для пользователей
- Заказы с распределёнными датами
- Инициализирует метрики аналитики

## Валидация тестовых данных

### validate_test_data

**Расположение**: `backend/core/management/commands/validate_test_data.py`

**Назначение**: Комплексная валидация всех тестовых данных в системе

**Параметры**:
- `--fix` - Исправить найденные ошибки (где возможно)
- `--model MODEL` - Валидировать только указанную модель
- `--export FILE` - Экспортировать результаты валидации в JSON файл
- `--detailed` - Показать детальную информацию о каждой ошибке

**Использование**:
```bash
# Валидация всех данных
python manage.py validate_test_data

# Валидация с исправлением ошибок
python manage.py validate_test_data --fix

# Валидация только питомцев
python manage.py validate_test_data --model=Pet

# Валидация с экспортом результатов
python manage.py validate_test_data --export=validation_results.json --detailed
```

**Проверяет**:
- Корректность JSON полей
- Целостность связей между моделями
- Соответствие данных форматам
- Наличие обязательных полей
- Валидность значений (цены, рейтинги, прогресс и т.д.)

**Модели для валидации**:
- `User` - Пользователи
- `Pet` - Питомцы
- `Product` - Товары
- `Course` - Курсы
- `Lesson` - Уроки
- `CoursePage` - Страницы курсов
- `ContentBlock` - Блоки контента
- `Order` - Заказы
- `OrderItem` - Элементы заказов
- `CartItem` - Элементы корзины
- `UserCourse` - Записи на курсы
- `Review` - Отзывы

## Структура тестовых данных

### Пользователи

**Staff пользователи**:
- Email: `admin@pitometsplus.ru`, `manager@pitometsplus.ru`, `support@pitometsplus.ru`
- Пароли: `admin123`, `manager123`, `support123`
- Права: `is_staff=True`, `is_superuser` (только для admin)

**Обычные пользователи**:
- Email: `{first_name}.{last_name}@test.ru`
- Пароль: `test123`
- Профили: имена, телефоны, города, даты рождения, биографии

### Питомцы

**Характеристики**:
- Виды: собаки и кошки
- Породы: реалистичные породы для каждого вида
- Вес: 3-50 кг для собак, 2-8 кг для кошек
- Возраст: от 6 месяцев до 15 лет
- Аллергии: 30% питомцев имеют аллергии
- Любимые продукты: 1-3 продукта

### Товары

**Категории**:
- Сухие корма для собак (4 шт.)
- Сухие корма для кошек (3 шт.)
- Влажные корма для собак (2 шт.)
- Влажные корма для кошек (3 шт.)
- Лакомства (3 шт.)

**Цены**: от 45 до 6500 рублей

### Курсы

**Категории**:
- Бесплатные курсы для собак (5 шт.)
- Платные курсы для собак (5 шт.)
- Бесплатные курсы для кошек (5 шт.)
- Платные курсы для кошек (5 шт.)
- Универсальные курсы (5 шт.)

**Цены**: от 0 (бесплатно) до 3990 рублей

### Заказы

**Характеристики**:
- Количество: 5-10 заказов на пользователя
- Статусы: pending, processing, shipped, delivered
- Типы доставки: standard, express, pickup
- Товары: 1-5 товаров в заказе
- Количество: 1-3 единицы каждого товара

### Записи на курсы

**Характеристики**:
- 60% пользователей записаны на курсы
- 1-3 курса на пользователя
- Прогресс: 0-100% (30% завершили курс)

## Рекомендации

### Перед созданием тестовых данных

1. **Создайте резервную копию**:
   ```bash
   python manage.py backup_database --output-dir backups/before_test_data
   ```

2. **Проверьте существующие данные**:
   ```bash
   python manage.py validate_test_data
   ```

### После создания тестовых данных

1. **Валидируйте созданные данные**:
   ```bash
   python manage.py validate_test_data --detailed
   ```

2. **Исправьте найденные ошибки**:
   ```bash
   python manage.py validate_test_data --fix
   ```

3. **Проверьте целостность данных**:
   ```bash
   python manage.py fix_orphaned_records --dry-run
   ```

### Регулярная валидация

1. **Добавьте в CI/CD pipeline**:
   ```yaml
   - name: Validate test data
     run: python manage.py validate_test_data --export=validation_results.json
   ```

2. **Настройте автоматическую валидацию**:
   ```bash
   # Еженедельная валидация
   0 2 * * 0 python manage.py validate_test_data --export=/path/to/results.json
   ```

### Очистка тестовых данных

Для очистки тестовых данных используйте Django shell или создайте отдельную команду:

```python
# В Django shell
from apps.users.models import User
from apps.shop.models import Product, Order
from apps.training.models import Course

# Удаление тестовых пользователей (кроме staff)
User.objects.filter(email__endswith='@test.ru').delete()

# Удаление всех заказов
Order.objects.all().delete()

# Удаление всех товаров и курсов (если нужно)
# Product.objects.all().delete()
# Course.objects.all().delete()
```
