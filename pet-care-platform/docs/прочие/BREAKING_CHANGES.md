## Обзор

Данный документ описывает все breaking changes, внесенные в процессе рефакторинга платформы "Питомец+".

### Что такое Breaking Changes?

Breaking changes - это изменения, которые могут нарушить работу существующего кода или требуют изменений в клиентских приложениях.

## Backend Breaking Changes

### Django 5.1.5

#### ✅ Обратно совместимо

Обновление с Django 4.2.8 на 5.1.5 является обратно совместимым для большинства случаев. Основные изменения:

- **JSONField**: Теперь использует `django.db.models.JSONField` (ранее `django.contrib.postgres.fields.JSONField`)
- **Timezone**: Рекомендуется использовать `USE_TZ = True`

**Действия**: Никаких действий не требуется, изменения обратно совместимы.

### Django REST Framework 3.15.2

#### ✅ Обратно совместимо

Обновление с DRF 3.14.0 на 3.15.2 является обратно совместимым.

**Действия**: Никаких действий не требуется.

### Сервисный слой

#### ⚠️ Изменения в структуре

Сервисы теперь наследуются от `BaseService` и используют `ServiceResult`:

**До**:
```python
class OrderService:
    @staticmethod
    def create_order(user, cart_items):
        # Прямое возвращение данных
        return order
```

**После**:
```python
class OrderService(BaseService):
    @staticmethod
    def create_order(user, cart_items):
        # Возвращение ServiceResult
        return ServiceResult.success(data=order)
```

**Действия**: Обновить код, использующий сервисы, для работы с `ServiceResult`.

### Обработка ошибок

#### ⚠️ Изменения в формате ошибок

Ошибки теперь возвращаются через `ApiError`:

**До**:
```python
return Response({'error': 'Message'}, status=400)
```

**После**:
```python
raise ApiError.bad_request('Message')
```

**Действия**: Обновить обработку ошибок в клиентском коде.

## Frontend Breaking Changes

### Lazy Loading

#### ⚠️ Изменения в импортах

Некоторые компоненты теперь загружаются лениво:

**До**:
```jsx
import UnifiedCheckout from './pages/Checkout/UnifiedCheckout'
```

**После**:
```jsx
const UnifiedCheckout = lazy(() => import('./pages/Checkout/UnifiedCheckout'))
```

**Действия**: Обновить импорты в `App.jsx`.

### Stores (Zustand)

#### ✅ Обратно совместимо

Изменения в stores являются обратно совместимыми. Добавлены новые утилиты, но старый код продолжает работать.

**Действия**: Рекомендуется использовать новые утилиты из `baseStore.js`.

### PropTypes

#### ✅ Обратно совместимо

Добавление PropTypes не нарушает существующий код.

**Действия**: Никаких действий не требуется.

## API Breaking Changes

### ✅ Обратно совместимо

Все изменения в API являются обратно совместимыми. Формат ответов не изменился.

### Изменения в формате ошибок

Формат ошибок стандартизирован, но старый формат также поддерживается:

**Новый формат**:
```json
{
  "error": "Сообщение об ошибке",
  "errors": {
    "field": ["Ошибка поля"]
  },
  "detail": "Детали ошибки"
}
```

**Действия**: Рекомендуется обновить обработку ошибок для использования нового формата.

## Миграция данных

### Архитектура курсов

#### ⚠️ Новая архитектура

Добавлена новая архитектура курсов на основе `CoursePage` и `ContentBlock`. Старая архитектура на основе `Lesson` продолжает работать.

**Действия**: 
- Для новых курсов рекомендуется использовать новую архитектуру
- Существующие курсы продолжают работать без изменений
- Миграция на новую архитектуру опциональна

### JSON поля

#### ✅ Обратно совместимо

Добавлена валидация JSON полей, но формат данных не изменился.

**Действия**: Никаких действий не требуется.
