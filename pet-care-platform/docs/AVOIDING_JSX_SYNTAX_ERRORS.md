# Инструкция: Как избежать ошибок синтаксиса JSX при рефакторинге

## Проблема

При рефакторинге React компонентов с условным рендерингом (ternary operators) часто возникают синтаксические ошибки из-за:
1. Неполного удаления старого кода
2. Несбалансированных скобок `()`, `{}`, `<>`
3. Оставшихся фрагментов ternary операторов (`: (`, `) : (`)
4. Дублирования блоков кода

## Типичные ошибки

### 1. Неполное удаление ternary оператора

**Плохо:**
```jsx
{hasChanges && (
  <button>Save</button>
) : (  // ← Ошибка: осталось от ternary
  <span>View</span>
)}
```

**Хорошо:**
```jsx
{hasChanges && (
  <button>Save</button>
)}
```

### 2. Дублирование компонентов

**Плохо:**
```jsx
<form>
  {/* View mode content */}
</form>
) : (
<form>  // ← Дубликат
  {/* Edit mode content */}
</form>
```

**Хорошо:**
```jsx
<form>
  {/* Single form with editable fields */}
</form>
```

### 3. Несбалансированные скобки

**Плохо:**
```jsx
{condition && (
  <div>Content</div>
)}  // ← Лишняя скобка
)}
```

## Правила безопасного рефакторинга

### Правило 1: Проверяй полную структуру

Перед изменением условного рендеринга найди **все** части конструкции:
```bash
# Найти все ternary операторы
grep -n ") : (" Component.jsx

# Найти все условные блоки
grep -n "&&" Component.jsx
```

### Правило 2: Используй поиск по паттернам

```bash
# Найти дублирующиеся теги
grep -n "<form" Component.jsx
grep -n "</form>" Component.jsx

# Убедись что количество открывающих = закрывающих
```

### Правило 3: Проверяй после каждого изменения

```bash
# Запусти компиляцию
npm run dev

# Или используй линтер
npx eslint Component.jsx
```

### Правило 4: При замене ternary на && удаляй обе ветки

**Было:**
```jsx
{isEditing ? (
  <EditForm />
) : (
  <ViewMode />
)}
```

**Нужно удалить:**
1. `{isEditing ? (`
2. `) : (`
3. `<ViewMode />`
4. `)}`

**Стало:**
```jsx
<EditForm />  // Или с условием: {isEditing && <EditForm />}
```

### Правило 5: Используй IDE форматирование

После изменений:
1. Сохрани файл
2. Примени автоформатирование (Shift+Alt+F в VS Code)
3. Если форматирование ломается — есть синтаксическая ошибка

## Чеклист перед коммитом

- [ ] Все ternary операторы имеют обе ветки или заменены на `&&`
- [ ] Количество `{` равно количеству `}`
- [ ] Количество `(` равно количеству `)`
- [ ] Количество открывающих тегов равно закрывающим
- [ ] `npm run dev` запускается без ошибок
- [ ] Страница рендерится в браузере

## Быстрая диагностика ошибок

| Ошибка | Вероятная причина |
|--------|------------------|
| `Unexpected token, expected ","` | Незакрытая скобка или лишний `) : (` |
| `Adjacent JSX elements must be wrapped` | Два элемента рядом без обёртки |
| `Unexpected token '<'` | JSX внутри обычного JS без скобок |
| `X is not defined` | Удалена переменная, но остались ссылки |

## Пример безопасного рефакторинга

### Задача: Убрать toggle между view/edit режимами

**Шаг 1:** Найти все части конструкции
```bash
grep -n "isEditing" Profile.jsx
```

**Шаг 2:** Определить структуру
```
Line 74: const [isEditing, setIsEditing] = useState(false)
Line 135: if (isEditing) { ... }
Line 335: {!isEditing ? (
Line 436: ) : (
Line 529: )}
```

**Шаг 3:** Удалить в правильном порядке
1. Удалить state: `const [isEditing, setIsEditing]`
2. Удалить ненужную ветку (view mode): строки 335-435
3. Удалить `) : (` на строке 436
4. Удалить `)}` на строке 529
5. Оставить нужную ветку (edit mode)

**Шаг 4:** Проверить
```bash
npm run dev
```
