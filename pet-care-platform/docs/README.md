# Документация Pet Care Platform

## Структура документации

Папка `docs/` содержит всю документацию проекта, организованную по тематическим категориям для удобства навигации и поиска.

### 📁 Структура папок

```
docs/
├── README.md                    # Этот файл - описание структуры документации
├── admin/                       # Документация по админ-панели
├── setup/                       # Установка и настройка проекта
├── development/                 # Разработка и архитектура
├── courses/                     # Курсы и образовательный контент
├── troubleshooting/             # Решение проблем и ошибки
├── deployment/                  # Развертывание и запуск
└── api/                         # API документация (пока пустая)
```

### 📋 Содержимое папок

#### `admin/` - Админ-панель
- `admin-user-stories-flow.md` - Поток пользовательских историй админ-панели
- `admin-redesign-context.md` - Контекст переделки админ-панели
- `admin-panel-context.md` - Общий контекст админ-панели

#### `setup/` - Установка и настройка
- `README_START.md` - Начало работы с проектом
- `README_LAUNCH.md` - Запуск проекта
- `БЫСТРЫЙ_СТАРТ.md` - Быстрый старт для backend
- `Виртуальное_окружение_для_начинающих.md` - Настройка виртуального окружения

#### `development/` - Разработка
- `PROJECT_STRUCTURE.md` - Структура проекта
- `стили окна регистрации моб вер.md` - Стили мобильной версии окна регистрации
- `Прикольная форма регистрации/` - Примеры форм регистрации (HTML/CSS/JS)

#### `courses/` - Курсы и обучение
- `FREE_COURSES_CONTENT.md` - Контент бесплатных курсов
- `COURSES_IMPLEMENTATION_CONTEXT.md` - Контекст реализации курсов

#### `troubleshooting/` - Решение проблем
- `AVOIDING_JSX_SYNTAX_ERRORS.md` - Избегание ошибок JSX синтаксиса
- `белый_экран_ошибка.md` - Решение проблемы белого экрана

#### `deployment/` - Развертывание
- `start_all.sh` - Скрипт запуска всего проекта
- `start-all.bat` - Пакетный файл запуска (Windows)
- `start-backend.bat` - Запуск backend (Windows)
- `start-frontend.bat` - Запуск frontend (Windows)
- `sync_after_pull.sh` - Синхронизация после pull

#### `api/` - API документация
*Пока пустая - планируется документация REST API проекта*

### 🎯 Как использовать документацию

1. **Для новых разработчиков**: Начните с папки `setup/` - там описаны шаги по установке и запуску проекта
2. **Для понимания архитектуры**: Читайте файлы из `development/`
3. **Для работы с админ-панелью**: Смотрите документацию в `admin/`
4. **При возникновении ошибок**: Проверяйте `troubleshooting/`
5. **Для развертывания**: Используйте материалы из `deployment/`
6. **Для работы с курсами**: Документация в `courses/`

### 📝 Рекомендации по структурированию

- **Новые документы** добавляйте в соответствующие тематические папки
- **Именование файлов**: Используйте английский язык с дефисами (kebab-case) для новых файлов
- **Описание изменений**: При добавлении новых документов обновляйте этот README.md
- **Категории**: Если документ не подходит ни под одну категорию, рассмотрите создание новой папки
