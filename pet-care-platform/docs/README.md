## Конвенции документирования

### Структура модуля

Каждый модуль в `modules/` содержит:

- `specification.md` — техническая спецификация (архитектура, API, модели, бизнес-логика)
- `user-stories.md` — пользовательские истории и критерии приёмки
- Дополнительные файлы при необходимости (algorithm.md, categories.md и т.д.)

### Формат specification.md

```markdown
# Модуль: Название

## Версия и статус
- Версия: X.Y
- Статус: Реализован / В разработке / Планируется

## Описание
Краткое описание функциональности.

## Архитектура
Компоненты, зависимости, схемы.

## API эндпоинты
Таблица эндпоинтов с методами, URL, описанием.

## Модели данных
Структура таблиц и полей.

## Бизнес-логика
Правила, алгоритмы, ограничения.
```

### Формат user-stories.md

```markdown
# User Stories: Название модуля

## US-XX: Название истории
- **Как**: [роль]
- **Хочу**: [действие]
- **Чтобы**: [ценность]

### Критерии приёмки
- [ ] Критерий 1
- [ ] Критерий 2
```
