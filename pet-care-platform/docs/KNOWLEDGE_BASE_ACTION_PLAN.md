## Краткое резюме

**Проблема:** База знаний о породах содержит ~60% дублирований, некорректные данные, отсутствует стандартизация параметров товаров.

**Решение:** Оптимизировать структуру, очистить дублирования, добавить детальные параметры питания в товары.

**Результат:** Полноценная персонализация магазина, курсов, PetID и дневника здоровья.

---

## Приоритетные задачи (MVP)

### ✅ Задача 1: Добавить `nutrition_params` в товары (КРИТИЧНО!)

**Срок:** 3 дня  
**Исполнитель:** Backend разработчик

**Что делать:**
1. Добавить поле `nutrition_params` (JSONField) в модель `Product`
2. Создать миграцию
3. Написать скрипт парсинга БЖУ из описаний товаров
4. Заполнить данные для топ-50 товаров вручную (образец)

**Структура `nutrition_params`:**
```json
{
  "age_group": "adult",
  "size_category": "medium",
  "protein_percent": 28.0,
  "fat_percent": 15.0,
  "calories_per_100g": 365,
  "ingredients": ["chicken", "rice", "corn"],
  "diet_type": "standard",
  "health_benefits": ["digestive_health"],
  "quality_class": "premium"
}
```

**Почему критично:** Без этих данных невозможен персонализированный подбор корма.

### ✅ Задача 2: Очистить дублирования в базе знаний

**Срок:** 5 дней  
**Исполнитель:** Data analyst + Backend

**Что делать:**

#### 2.1. Вынести общие процедуры ухода
- Создать `data_breeds/common/common_care_procedures.json`
- Перенести туда общие процедуры (чистка зубов, когтей, лотка)
- Удалить дублирования из `breed_care.json`
- Оставить только породоспецифичные процедуры

**Экономия:** ~12k строк (80% от `breed_care.json`)

#### 2.2. Вынести общие вакцинации
- Создать `data_breeds/common/common_vaccinations_dogs.json`
- Создать `data_breeds/common/common_vaccinations_cats.json`
- Удалить дублирования из `breed_medical.json`

**Экономия:** ~33k строк (95% от `breed_medical.json`)

#### 2.3. Удалить общие болезни
- Удалить из `breed_health.json`: паразиты, ожирение (если не специфично для породы)
- Оставить только генетические и породоспецифичные заболевания

**Экономия:** ~11k строк (40% от `breed_health.json`)

**Общая экономия:** ~56k строк (~50% объема), ~5 MB

### ✅ Задача 5: Реализовать алгоритм подбора корма

**Срок:** 5 дней  
**Исполнитель:** Backend разработчик

**Что сделать:**

1. Создать сервис `FoodRecommendationService`
2. Реализовать функции:
   - `calculate_daily_calories(pet, breed_data)` - расчет калорий
   - `calculate_macro_needs(pet, breed_data)` - расчет БЖУ
   - `filter_suitable_foods(pet, breed_data)` - фильтрация кормов
   - `rank_foods(foods, pet)` - ранжирование по релевантности
   - `calculate_portions(pet, food)` - расчет порций

3. Создать эндпоинт:
```python
POST /api/pets/{pet_id}/recommend-food/
Response: {
    "recommended_foods": [...],
    "daily_calories": 1000,
    "macros": {"protein": 70, "fat": 17, "carbs": 142},
    "daily_portion": 250,
    "meals_per_day": 2,
    "portion_per_meal": 125
}
```

**Документация:** См. `docs/FOOD_RECOMMENDATION_ALGORITHM.md`
