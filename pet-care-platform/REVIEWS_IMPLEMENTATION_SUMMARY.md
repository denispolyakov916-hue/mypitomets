# Сводка реализации системы рейтингов и отзывов

## Обзор

Реализована полная система рейтингов и отзывов для товаров и курсов на фронтенде с использованием передовых практик UX/UI для e-commerce. Система позволяет пользователям просматривать рейтинги, оставлять отзывы только после покупки, и управлять своими отзывами.

## Функциональность

### 1. Отображение рейтинга

✅ **На карточках товаров и курсов:**
- Показывается средний рейтинг (0-5 звезд)
- Отображается количество отзывов
- Автоматически скрывается, если данных нет

✅ **На страницах деталей:**
- Рейтинг в заголовке страницы
- Визуализация распределения рейтингов
- Статистика по оценкам (1-5 звезд)

### 2. Система отзывов

✅ **Просмотр отзывов:**
- Список всех отзывов с пагинацией
- Информация о пользователях (маскированные имена)
- Время публикации в относительном формате
- Бейдж "Подтвержденная покупка"

✅ **Создание отзыва:**
- Доступно только после покупки товара/курса
- Выбор рейтинга (1-5 звезд)
- Текстовый комментарий (10-2000 символов)
- Валидация формы

✅ **Редактирование отзыва:**
- Возможность обновить свой отзыв
- Сохранение истории редактирования
- Индикатор отредактированного отзыва

### 3. Проверка прав доступа

✅ **Автоматическая проверка:**
- Проверка покупки через API
- Показ формы только для купивших
- Информативные сообщения для не купивших

## API интеграция

### Ожидаемые эндпоинты (должны быть реализованы на бекенде):

1. **GET** `/api/reviews/products/{id}/` - Получение отзывов товара
2. **GET** `/api/reviews/courses/{id}/` - Получение отзывов курса
3. **POST** `/api/reviews/products/{id}/` - Создание отзыва на товар
4. **POST** `/api/reviews/courses/{id}/` - Создание отзыва на курс
5. **PUT** `/api/reviews/products/{id}/reviews/{review_id}/` - Обновление отзыва
6. **PUT** `/api/reviews/courses/{id}/reviews/{review_id}/` - Обновление отзыва
7. **DELETE** `/api/reviews/{type}/{id}/reviews/{review_id}/` - Удаление отзыва
8. **GET** `/api/reviews/{type}/{id}/eligibility/` - Проверка возможности отзыва

### Обновление существующих эндпоинтов:

1. **GET** `/api/shop/products/{id}/` - Добавить поля `rating`, `reviews_count`, `is_purchased`
2. **GET** `/api/courses/{id}/` - Добавить поля `rating`, `reviews_count`

## Логика работы

### 1. Отображение рейтинга

1. При загрузке товара/курса API возвращает `rating` и `reviews_count`
2. Компонент `Rating` отображает звезды и количество отзывов
3. Если данных нет, компонент не отображается

### 2. Секция отзывов

1. При монтировании загружаются отзывы и проверяется возможность оставить отзыв
2. Если пользователь купил товар/курс, показывается форма отзыва
3. Если отзыв уже существует, форма показывается в режиме редактирования
4. После создания/обновления отзыва список автоматически обновляется

### 3. Проверка покупки

1. При загрузке страницы деталей API возвращает `is_purchased` (для товаров) или `is_owned` (для курсов)
2. Это значение передается в `ReviewsSection`
3. Дополнительная проверка через API `/eligibility/` для точной валидации

## Обработка ошибок

### Сценарии обработки:

1. **API недоступен:**
   - Показывается сообщение об ошибке
   - Компоненты не ломают страницу
   - Возможность повторить запрос

2. **Пользователь не купил товар/курс:**
   - Показывается информативное сообщение
   - Форма отзыва не отображается

3. **Ошибки валидации:**
   - Подсветка полей с ошибками
   - Четкие сообщения об ошибках
   - Предотвращение отправки невалидных данных

4. **Ошибки прав доступа:**
   - Сообщение о необходимости покупки
   - Информативные уведомления

## Техническое задание для бекенда

Подробное техническое задание с примерами запросов и ответов находится в файле:

**`REVIEWS_AND_RATINGS_TZ.md`**

### Основные требования:

1. **Модель Review** с полями:
   - `review_type` (product/course)
   - `product` или `course` (ForeignKey)
   - `user` (ForeignKey)
   - `rating` (1-5)
   - `comment` (текст, 10-2000 символов)
   - `is_verified_purchase` (boolean)
   - `is_approved` (boolean, для модерации)

2. **API эндпоинты:**
   - GET/POST `/api/reviews/products/{id}/`
   - GET/POST `/api/reviews/courses/{id}/`
   - PUT/DELETE `/api/reviews/{type}/{id}/reviews/{review_id}/`
   - GET `/api/reviews/{type}/{id}/eligibility/`

3. **Логика проверки покупки:**
   - Для товаров: проверка заказов со статусом `delivered`/`shipped`
   - Для курсов: проверка `UserCourse` записей

4. **Обновление существующих эндпоинтов:**
   - Добавить `rating`, `reviews_count`, `is_purchased` в ответы
