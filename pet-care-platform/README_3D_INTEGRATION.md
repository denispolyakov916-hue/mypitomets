## 🎯 Что это делает

### Проблема
Blender использует сложные процедурные материалы и частицы для создания "шерсти". Веб-браузеры не могут их отображать напрямую.

### Решение
**Запекание (Baking)** - преобразование сложных материалов в простые текстуры:

```
┌──────────────────┐
│ Blender File     │  F-5.0.blend (116MB)
│                  │  • Hair particles
│ [Процедурная     │  • Shader nodes
│  шерсть]         │  • Complex materials
└────────┬─────────┘
         │
         │ 🔥 ЗАПЕКАНИЕ
         │
         ▼
┌──────────────────┐
│ Текстуры         │  Bakes (PNG файлы)
│                  │  
│ ├─ basecolor.png │  ← Цвет + ворс
│ ├─ normal.png    │  ← Объём/рельеф
│ ├─ roughness.png │  ← Блеск поверхности
│ └─ ao.png        │  ← Тени/глубина
└────────┬─────────┘
         │
         │ 📦 ЭКСПОРТ
         │
         ▼
┌──────────────────┐
│ GLB File         │  assistant.glb (7MB)
│                  │  
│ ├─ Geometry      │  ← Mesh + UV
│ ├─ Textures      │  ← Embedded PNG
│ ├─ Materials     │  ← PBR материал
│ └─ Animations    │  ← Idle, wave, etc.
└────────┬─────────┘
         │
         │ 🌐 WEB
         │
         ▼
┌──────────────────┐
│ Браузер          │  http://localhost:5173/profile
│                  │  
│ [3D модель с     │  • Интерактивная
│  детальной       │  • 60 FPS
│  шерстью]        │  • Анимированная
└──────────────────┘
```

### Шаг 2: Обновление версии ⚡

После успешного экспорта **обязательно** обновите версию:

**Файл:** `frontend/src/components/Assistant3D.jsx`

**Было:**
```javascript
const MODEL_URL = '/models/assistant.glb?v=8'
```

**Стало:**
```javascript
const MODEL_URL = '/models/assistant.glb?v=9'
```

**Зачем это нужно?**
- Браузеры кэшируют GLB файлы
- Без изменения версии пользователи увидят старую модель
- `?v=9` заставляет браузер загрузить новый файл

## 🎬 Результат работы

### Что вы получите

#### 📦 GLB файл
- **Расположение:** `frontend/public/models/assistant.glb`
- **Размер:** ~7MB
- **Содержит:**
  - Геометрию (mesh с UV-развёрткой)
  - Материалы (PBR Principled BSDF)
  - Текстуры (embedded PNG)
  - Анимации (idle, wave, и др.)

#### 🖼️ Текстуры (в папке bakes)
Для каждого материала 4 текстуры:

1. **basecolor.png** (2048×2048)
   - Основной цвет
   - Имитация ворса через шум
   - Color space: sRGB

2. **normal.png** (2048×2048)
   - Карта нормалей
   - Создаёт иллюзию объёма
   - Color space: Non-Color

3. **roughness.png** (2048×2048)
   - Шероховатость поверхности
   - Влияет на отражения
   - Color space: Non-Color

4. **ao.png** (2048×2048)
   - Ambient Occlusion
   - Затемняет щели и складки
   - Color space: Non-Color

#### ⚛️ Интерактивный компонент

**Визуальные эффекты:**
- 🌅 HDRI окружение (sunset)
- 💡 Многослойное освещение
- 🌗 Реалистичные тени
- ✨ Tone mapping (ACESFilmic)

**Интерактивность:**
- 🖱️ Следит за курсором
- 🤚 Машет при клике
- 💓 Пульсирует при клике
- 🌊 Плавное покачивание

**Производительность:**
- 📦 Preloading (быстрая загрузка)
- 🔄 Кэширование (версионирование)
- 🎯 60 FPS на средних ПК

## ❓ FAQ

### Q: Сколько времени занимает процесс?
**A:** 5-15 минут в зависимости от:
- Сложности модели
- Количества материалов
- Разрешения текстур (2048×2048)
- Мощности ПК (GPU ускоряет процесс)

---

### Q: Какого размера будет результат?
**A:** 
- GLB файл: 5-10MB
- Текстуры (bakes): 5-15MB суммарно
- Итого: ~15-20MB

**Оптимизация:**
- Уменьшить разрешение текстур (2048 → 1024)
- Использовать Draco compression
- Конвертировать PNG в WebP

---

### Q: Можно ли использовать для других моделей?
**A:** Да! Скрипт универсальный:
1. Замените путь к blend файлу в `bake_fur_model.py`:
   ```python
   BLEND_FILE = "/путь/к/вашей/модели.blend"
   ```
2. Запустите `./scripts/bake_and_export.sh`

---

### Q: Что если модель выглядит чёрной?
**A:** Это означает проблему с текстурами:
1. Проверьте что текстуры embedded: `./scripts/test_model.sh`
2. Повторите запекание: `./scripts/bake_and_export.sh`
3. Убедитесь что используется Cycles (не Eevee)

---

### Q: Что если низкий FPS?
**A:** Оптимизируйте:
1. Уменьшить разрешение текстур (1024 вместо 2048)
2. Упростить освещение (убрать Environment)
3. Уменьшить полигоны (Decimate modifier)
4. Использовать Draco compression

---

---

### Q: Модель не обновляется после замены
**A:** Причины:
1. **Не обновлена версия** - увеличьте `?v=9` → `?v=10`
2. **Кэш браузера** - очистите `Ctrl+Shift+R`
3. **Frontend не перезапущен** - `npm run dev`
4. **Файл не заменился** - проверьте дату модификации

---

### Q: Можно ли запустить без Blender UI?
**A:** Да! Используйте `bake_and_export.sh`:
```bash
./scripts/bake_and_export.sh
```
Blender запустится в фоновом режиме (headless).

---

### Q: Где посмотреть детальные логи?
**A:** 
- **Blender:** Window → Toggle System Console
- **Bash скрипт:** Вывод в терминал
- **Браузер:** F12 → Console

---

### Q: Что делать если процесс завис?
**A:**
1. Подождите 15-20 минут (особенно на слабом ПК)
2. Если не помогло - `Ctrl+C` и перезапустите
3. Проверьте логи на ошибки
4. Уменьшите TEXTURE_SIZE в скрипте (2048 → 1024)

## 🛠️ Системные требования

### Для запекания (Blender)
- ✅ Blender 3.0+ (у вас: 5.0.1)

- React 18+
- Three.js 0.160+
- React Three Fiber 8+
- @react-three/drei 9+
- Браузер с WebGL 2.0 (Chrome, Firefox, Safari, Edge)
